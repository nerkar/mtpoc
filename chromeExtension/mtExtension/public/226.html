Step 2. Define the Abend-AID Web Browser Interface URL
Based on the options specified for the above viewing server parameters, define a site-
specific URL for Abend-AID Web Browser Interface access that you make known to your
site’s users in the following format:
http://host_name:TCP_PORT/AAVIEW.HTML
or
http://IP_address:TCP_PORT/AAVIEW.HTML
For example,
http://cw01:48226/AAVIEW.HTML
or
http://nn.nn.nn.nnn:48226/AAVIEW.HTML
.
Abend-AID Plug-in for Eclipse Considerations
Notes:
1.
The Abend-AID plug-in for Eclipse requires Eclipse Release 3.4 or more current.
2.
The Abend-AID viewing server associated with this Abend-AID Web Browser Interface
must be active on the mainframe in order for you to access reports and diagnostics.
See “Configuring Abend-AID Web Browser Interface Access” on page 16-20 for more
information.
Using the Abend-AID plug-in for Eclipse allows you to start the Abend-AID Web Browser
Interface from within Eclipse. If you do not have Eclipse installed, you can continue to
start the Abend-AID Web Browser Interface using your site-defined URL. See “Step 2.
Define the Abend-AID Web Browser Interface URL” on page 16-22 for more information.
Refer to the Abend-AID Web Browser Interface
online help
for detailed information about
using this interface for viewing Abend-AID reports and SVC dump diagnostics.
Beginning with Release 12.1, the Abend-AID plug-in for Eclipse has been removed from
the Abend-AID product media and is now available on the Topaz Workbench (formerly,
Compuware Workbench) product media. For the most current Abend-AID plug-in for
Eclipse installation and user information, refer to the Topaz Workbench FrontLine
website at
http://go.compuware.com
.
