Abend-AID Installation: Generated DB2 Support Screen
Note:
You can change the displayed plan name. However, Compuware recommends that
you use the same plan name for all DB2 releases and subsystems within the
Abend-AID for CICS installation.
Do one of the following on the Abend-AID Installation: Generated DB2 Support
screen shown in Figure 6-5:
–
If this the first time you are using the Abend-AID Release 16.05 installation
dialog to install support for any DB2 release, you must first enter your site’s
Abend-AID for CICS plan name if it is different from the displayed default. Use
the I (Insert) command to display the Abend-AID Installation: Site-specific DB2
Support screen and complete this panel with the information for at least one
DB2 release and subsystem for the plan name as shown in Figure 6-6 on page
6-10. Then you can use the END key (PF3) to return to the screen displayed in
Figure 6-5 on page 6-9 and use the E (Edit), I (Insert), R (Replicate), or D (Delete)
line commands to update the installation dialog information for all of the DB2
versions for each DB2 subsystem required at your site.
–
If you have already installed support for at least one DB2 release, subsystem, and
plan name, then the DB2 summary information for each DB2 release already
installed appears on the Abend-AID Installation: Generated DB2 Support screen
in Figure 6-5. Use the E (Edit), I (Insert), R (Replicate), or D (Delete) line
commands to update the installation dialog information and to display the
Abend-AID Installation: Site-specific DB2 Support screen shown in Figure 6-6 on
page 6-10. You can update/add information on this panel at any time for any
DB2 version for any DB2 subsystem required at your site.
Enter the GENERATE command on the Abend-AID Installation: Site-specific DB2
Support screen (Figure 6-6 on page 6-10) to generate the JCL to install DB2 support
for each release, subsystem, and plan name. Each JCL member generated into the
installation sample (CTL) library is $21
nxxxx
, where
n
represents the one-numeric
character DB2 release identifier and
xxxx
is the four-alphanumeric character DB2
subsystem ID. The JCL is replaced in the installation sample (CTL) library only if the
DB2 release and subsystem are the same.
