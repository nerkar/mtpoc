Abend-AID Installation and Customization Guide
Automatically Allocated Dump Datasets
z/OS provides a facility to take SDUMPs directly to user-defined datasets, bypassing
SYS1.DUMPxx. The Abend-AID Viewer can be customized to automatically import and
analyze these “automatically allocated” dump datasets.
For this support, you must do the following:
1.
Install the MVS post-dump exit, as described in “MVS Post-Dump Exit” on page 17-3.
2.
Modify your SVC dump capture profiles to indicate which SVC dumps are to be taken
to automatically allocated datasets, imported, and analyzed. SVC dump capture
profiles are described in “Modifying Region Dump Profiles” on page 20-14.
Note:
On the Region Dump Profile screen, specify a dump type of SYS1 for
automatically allocated dump datasets. The Abend-AID Viewer recognizes
that SYS1.DUMPxx has actually been bypassed, determines the name of the
automatically allocated dataset, and imports it.
Console Dumps
To invoke automatic import for a console dump, do the following:
1.
Install the MVS post-dump exit, as described in “MVS Post-Dump Exit” on page 17-3.
2.
Use the following MVS system command to invoke the console dump:
DUMP COMM=(AAVWR DUMP: SYSTEM=mvssys,JOB=jjjjjjjj)
The dump title must be at least AAVWR DUMP: SYSTEM=mvssys. The AAVWR DUMP
portion of the title is sufficient to cause the MVS post-dump exit to start the
automatic import procedure. For the Abend-AID Viewer to find a matching SVC
dump profile entry, the SYSTEM=mvssys portion must be provided, where mvssys is
the APPLID of an MVS system. You must also specify the job name as part of the
dump title.
Using Automated Operations Facilities
If you are using an automated operations facility to copy SYS1.DUMPxx datasets after
dumps are written to them, and you want to automatically import those copied datasets,
the MVS post-dump exit is not required. You can import copied dumps using the MVS
MODIFY command with the Abend-AID Viewer IMPORT keyword. The automated
operations tool can issue this command. For a complete description of the MVS MODIFY
command for the viewing server, refer to Appendix A, “Controlling the Abend-AID
Viewing Server, BDCAS, and TDCAS”.
If you install the MVS post-dump exit and want your automated operations package to
recognize actions taken against SVC dumps by the post-dump exit, you can use the
following Abend-AID Viewer messages to determine the status of dump processing:
•
MSDSD3045I:
Dataset-name has been copied successfully.
•
MSDSD3046I:
Dataset-name has been scheduled for automatic import into server
region.
Duplicate Dump Suppression
The Abend-AID Viewer does not provide a facility to suppress duplicate SVC dumps.
Instead, Compuware recommends that you use the MVS Dump Analysis and Elimination
(DAE) facility to suppress or eliminate duplicate SVC dumps.
