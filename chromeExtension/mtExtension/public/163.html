ARM_ELEMENT_SUFFIX=tdcasname|suffix
Specifies the one- to eight-character suffix of the ARM element name used to register
with ARM. The element name used by Abend-AID for CICS to register with ARM is
KMPssidsuffix, where
KMP
is a constant,
ssid
is the four-character MVS system ID, and
suffix
is the ARM_ELEMENT_SUFFIX. The element name is used by ARM to select a
restart policy. Each Abend-AID for CICS TDCAS using ARM requires a unique suffix.
Unless you specify a different suffix, the TDCAS uses the default, the TDCAS name
parameter value, which is the one- to eight-character TDCAS name specified in the
TDCAS execute statement.
Note:
If the automatic restart manager (ARM) restarts a job, JES discards all non-spin
SYSOUT datasets created during the previous execution. SPIN=UNALLOC is
automatically included on the //FDBDLOG DD statement for the SYSOUT
datasets in the TDCAS generated JCL to avoid losing valuable debugging
information. For example:
//FDBDLOG DD SYSOUT=*,SPIN=UNALLOC
Modifying the TDCAS JCL ($67TDCAS)
Installation sample library (CTL) member $67TDCAS contains sample startup JCL for the
Abend-AID subsystem and transaction dump capture address space. The Abend-AID
subsystem is defined by the MVS_SUBSYSTEM viewing server parameter, as described in
“MVS_SUBSYSTEM=(name,OWNER|USER)” on page 15-4. Review and modify the sample
JCL according to the guidelines in this section.
The job has two steps. The first step starts the Abend-AID subsystem. The second step
starts the transaction dump capture address space. You need only one Abend-AID
subsystem regardless of how many TDCASs you run on the MVS image. If the Abend-AID
subsystem is already active when you submit this job, the step to start the subsystem is
automatically skipped.
1. Modify the JOB or PROC Statements
You can run the transaction dump capture address space (TDCAS) as either a job or a
started task. If you want to run the TDCAS as a started task, change the JOB statements to
PROC statements, and modify the TDCAS JCL according to the instructions in the
following sections. Then, copy the PROC to a library in your system PROCLIB
concatenation.
Note:
Compuware strongly recommends that you run the TDCAS as a started task and
that you start it at IPL. The TDCAS must be active for Abend-AID for CICS to
capture transaction dumps.
If you run the transaction dump capture address space as a job, modify the TDCAS JCL
according to the instructions in the following sections. Then, submit the TDCAS JCL
when you want to start the TDCAS.
Notes:
1.
The TDCAS JCL is always generated with a TIME=1440 parameter on the execute
statement to prevent the TDCAS from timing out, regardless of the amount of CPU
time it uses. If necessary, modify this parameter to your site’s requirements.
2.
The TDCAS region size defined on the execute statement is 8 megabytes. This is the
minimum recommended region size for the TDCAS.
3.
For z/OS 1.9 and more current, the TDCAS and CICS region may require 64-bit
storage during CICS transaction dump capture. You can do this using the
MEMLIMIT=nnnnM parameter on the JOB or EXEC statements in your TDCAS JCL
