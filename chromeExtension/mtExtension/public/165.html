5. Verify the TDCAS Log Destination
The //FDBDLOG DD statement specifies a SYSOUT class that is used to record TDCAS
informational and error messages. Compuware recommends that you check this output if
you experience problems using Abend-AID for CICS, since many TDCAS-related messages
are written there.
The default value is
SYSOUT=*, SPIN=UNALLOC
Note:
SPIN=UNALLOC is
required
to prevent the loss of JES output with the
ARM_ENABLED=YES parameter setting in the viewing server/TDCAS. However,
this default setting is also valid for sites not using the ARM restart option.
6. ABNLIGNR DD Statements
The TDCAS JCL contains an //ABNLIGNR DD DUMMY statement so that batch Abend-
AID does not process Abend-AID for CICS TDCAS dumps. These statements should not be
removed from the TDCAS JCL.
Displaying List of CICS Regions Registered with TDCAS
To display the list of CICS regions registered with the TDCAS, log onto the TDCAS.
Display the Active CICS Regions screen by either selecting it from the TC Primary
Options menu or by entering the LISTCICS fast-path command in the COMMAND (or
Option) field on any TDCAS screen. The Active CICS Regions screen lists all of the CICS
regions currently registered with the TDCAS and enables you to recycle Abend-AID for
CICS in any or all of the registered CICS regions.
