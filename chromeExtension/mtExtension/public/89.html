Binding the DBRMs
The Abend-AID for CICS installation sample library (CTL) contains members
beginning with $
21nxxxx
, where
n
represents the one-numeric character DB2 release
identifier and
xxxx
is the four-alphanumeric character DB2 subsystem name. These
jobs bind the DB2 DBRMs for each DB2 subsystem supported by Abend-AID for CICS
at your site. You indicated which DB2 releases to support by specifying the names of
the corresponding DB2 load libraries in the installation dialog.
Beginning with Release 12.1, the default Abend-AID for CICS DB2 option plan name
is
CWDB2PLN.
You can change the plan name, but it must match the name specified
in the RDO entry (see “Adding the DB2 RDO Entry” on page 5-8). Sample DBRMs are
provided in the Abend-AID target sample library (SKAZSAMP) members ACDB
nnn
.
These members are bound into packages prior to binding plan CWDB2PLN.
The $21
nxxxx
jobs are tailored based on the information you specified in the
installation dialog. The specific $21
nxxxx
members generated correspond to the
releases of DB2 that you specified in the installation dialog.
Run all $21
nxxxx
jobs in the installation sample library (CTL) to fully support
your releases of DB2.
You can still run the Abend-AID for CICS DB2 option without
binding the DBRMs, but much of the information provided by the option is missing
from the Abend-AID for CICS displays if you do not perform the bind.
Notes:
a.
If you do not run $21
nxxxx
for a DB2 subsystem, program CTCCDB25 will
experience a -922 abend. This abend appears as an entry on the Abend-AID
Directory for the AADB transaction.
a.
Abend-AID for CICS supports multiple subsystems, but the DBRMs must be
bound for each subsystem.
