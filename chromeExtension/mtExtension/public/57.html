4. Import Existing Routing and/or Global Options Data
Run the following jobs if you want to import your existing routing and/or global options
data from a previous release of batch Abend-AID for use with Release 16.05.
•
Execute installation library (CTL) JCL member $62PROUT to import batch Abend-
AID routing information from a previous release of Abend-AID. Job $62PROUT is
created only if you specified an Abend-AID customization load library containing a
CWROUTE module in Figure 4-2 on page 4-2 of the installation dialog. If $62PROUT
was not created, the Abend-AID customization file contains only the default routing
option of routing all Abend-AID reports to the site default report shared directory.
•
Execute installation library (CTL) JCL member $62PGLOB to import batch Abend-
AID global option information from a previous release of Abend-AID. Job $62PGLOB
is created only if you specified an Abend-AID customization load library containing a
CWGLOBAL module in Figure 4-3 on page 4-2 of the installation dialog. If
$62PGLOB was not created, the Abend-AID customization file contains only the
batch Abend-AID global option installation default values.
Alternatively, you can use the Release 16.05 batch Abend-AID online customization
options in the Abend-AID Viewer to create/modify report routing criteria and modify the
global options. See “Accessing the Batch Customization Options” on page 20-37 for more
information.
5. Execute JCL Member $11VERIF
Abend-AID installation library (CTL) JCL member $11VERIF executes the utility
CWVERIFY to verify that the batch Abend-AID load library was correctly built.
Review and save for future reference the report that CWVERIFY provides. It includes the
following sections:
•
List of Batch Abend-AID Modules Linked:
Missing modules are noted with an error
message. If the first column of the report is blank, the module was loaded correctly. A
message in the first column may indicate a problem in linking that module. If so,
refer to the
Should Be
column for the correction required.
•
List of Batch Abend-AID Modules — Recap:
Review the recap to determine if any
modules were loaded with errors. If an error is indicated, check the
Should Be
column in the “List of Abend-AID Modules Linked” section, and then correctly relink
the modules.
•
IBM-Dependent Modules:
Batch Abend-AID uses IGWASMS to provide Data Facility
Storage Management Subsystem (DFSMS) support and must be available to Abend-
AID at abend time. If CWVERIFY cannot find IGWASMS, the message MODULE
IGWASMS NOT FOUND AND IS REQUIRED FOR Abend-AID SMS SUPPORT is
displayed in the Recap section. If your site is not using DFSMS, you can ignore this
message. See “IBM DFSMS Modules” on page 11-6 for more information on Abend-
AID’s DFSMS support.
•
