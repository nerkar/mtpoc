Controlling the Abend-AID Viewing Server, BDCAS, and TDCAS
A-3
Closing and Reopening the FDBDLOG File
The Abend-AID Viewer LOGSPOOL command allows you to close and reopen the viewing
server, BDCAS, or TDCAS FDBDLOG file. The next time you issue the LOGSPOOL command,
you can print, delete, or offload the closed file. Executing this command can reduce the
amount of spool space consumed by the viewing server, BDCAS, or TDCAS in continuous
operation environments.
Notes:
1.
You can restrict access to this command by coding the LOGSPOOL subparameter of
the EXTERNAL_SECURITY_FUNCTION_CHECK viewing server configuration
parameter. Refer to “Parameters to Enable External Security for Viewing Servers” on
page 15-8 for more information.
2.
To free the initial FDBDLOG file, stop the viewing server, BDCAS, or TDCAS. Modify
the FDBDLOG DD statement to indicate a specific SYSOUT class other than asterisk
(*)
and to include the FREE=CLOSE parameter. Restart the viewing server, BDCAS, or
TDCAS.
Viewing Server, BDCAS, and TDCAS MODIFY
Commands
The MVS MODIFY command allows you to perform the following functions:
•
Stop the viewing server, BDCAS, or TDCAS
•
Import a dump
•
Display viewing server, BDCAS, or TDCAS information
•
Close and reopen the viewing server or TDCAS FDBDLOG file.
•
Controlling opening and closing of load libraries for CSECT processing in the
TDCAS.
Stopping the Viewing Server, BDCAS, or TDCAS
The MVS MODIFY command stops the viewing server, BDCAS, or TDCAS from the MVS
console.
The following syntax is valid for the MODIFY command:
jobname
The viewing server, BDCAS, or TDCAS job name.
nnnn
Timer value that specifies the number of seconds to elapse before the current user
and transaction load close down. The default value is 60 seconds.
