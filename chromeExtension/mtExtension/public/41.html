Notes:
1.
You can run Abend-AID for CICS Release 16.05 concurrently with a prior release.
2.
Each CICS region converts to Release 16.05 when the load libraries are included in
the startup JCL. Until then, the regions can continue to use the previous release.
3.
The Release 16.05 transaction dump capture address space (TDCAS) region is
compatible with CICS regions running previous releases of Abend-AID for CICS.
However, the CICS release must be a supported level as determined by the TDCAS.
4.
After the TDCAS has been converted to Abend-AID for CICS Release 16.05, migration
of the CICS regions to Release 16.05 can be delayed until it is convenient to recycle
them.
Migrating Abend-AID Database Support
Follow these steps to update to Abend-AID database support for Release 16.05:
1.
For DB2 support:
a.
Generate the DB2 configuration and customization JCL by following the
procedure in Chapter 6, “Database Support Configuration” that uses your input
from the DB2 Support option dialog panels. Enter the
generate
command on the
last dialog panel.
b.
Provide the generated DB2 installation JCL to your site’s DBA(s) to run so that
the Abend-AID plans are bound and authority is granted.
2.
For CA-IDMS support, complete the procedure in “Configuring Batch Abend-AID for
IDMS” on page 6-14.
Cloning Abend-AID Libraries to Other LPARs
If you plan to install Release 16.05 on other LPARs in your site’s sysplex, you can use
installation sample library (CTL) job COPYLIBS to allocate and copy the Abend-AID
libraries. Before executing this job, depending on how your product files were delivered,
you must have already executed the SMP/E installation jobs as described in the
Compuware Installer Mainframe Products SMP/E Installation Guide
. Sites installing batch Abend-
AID must have executed the $62CUSTL job as described in the “1. Allocate the Abend-
AID Customization Load Library ($60CUSTL)” on page 3-6.
