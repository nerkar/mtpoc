CACHE_CSECT=(YES|no,size)
Specifies whether the CSECT caching support for the TDCAS is enabled. If enabled,
the second parameter specifies the number of entries in the caching table.
With CSECT caching support enabled, CSECT information for abending programs is
stored in a cache table. If the same CSECT then participates in a later abend, the
CSECT information is retrieved from the cache table, thereby improving dump
capture performance.
YES
CSECT caching support is enabled. This value is the default.
NO
CSECT caching support is disabled.
size
This value is the number of entries available in the cache table to support the CSECT
caching function. Valid values are 32 through 10240. The default value is
256
.
Approximately 100 bytes is used for each table entry. Using the default value of 256,
the caching table uses about 25K of storage.
HEALTH_CHECK=YES|no
Specifies whether health checks using the IBM Health Checker for z/OS are started
when the TDCAS is started. The health checks provide information about storage
utilization and DB2 connections. The default is
YES
. See Appendix H, “Abend-AID
Health Checker” for more detailed information.
WRITE_SMF=(NO|yes,SMF record number)
Specifies whether to write out SMF records containing information for each abend at
the time of dump capture for CICS reporting. Customers with a suitable reporting
process can use these records to produce reports on abend activity. This parameter
has been provided to enable the writing of these SMF records and to allow the user to
specify the SMF record number.
NO
Do not write an Abend-AID for CICS SMF dump capture record. This is the
default.
YES
Write an Abend-AID for CICS SMF dump capture record.
SMF record number
The SMF record number to be used by Abend-AID for CICS. This number can be
any available record number between 128 and 255, but it must not be the same
number that is used by any other z/OS software.
Member SMFDSECT in the Abend-AID installation sample library (CTL) contains an
Assembler DSECT layout for the Abend-AID for CICS SMF record.
Parameters Required to Support MVS Automatic Restart Manager
Abend-AID can register to be restarted by the MVS automatic restart manager (ARM). The
following parameters are required for Abend-AID for CICS transaction dump capture
address spaces to be restarted by ARM in conjunction with the site-defined ARM policies.
ARM_ENABLED=NO|yes
Specifies whether the transaction dump capture address space is to be registered for
restart with ARM. The default is
NO
.
