Abend-AID Installation and Customization Guide
Configuring Batch Abend-AID for IDMS
Batch Abend-AID for IDMS supports CA-IDMS Releases 18.0, 18.5, and 19.0.
Note:
Be sure to complete steps 1 through 3 of the configuration procedures that follow.
Abend-AID for IDMS is not operable without them.
Complete the following steps to configure Abend-AID for IDMS. After upgrading to
Release 18.0 or more current from a release before 12.00, relink all application modules
that include CA-IDMS modules.
1. Select the IDMS Support option.
When you select the second option, IDMS Support, from the Abend-AID Installation:
Site-specific Database Support menu (Figure 6-2 on page 6-2), the Abend-AID
Installation: Site-specific IDMS Support screen is displayed as shown in Figure 6-7.
Figure

2. Complete the screen fields
Specify the desired options, the IDMS load library dataset name, and the job card
information.
Enter
generate
to create the $$IDMLOG JCL.
3. Run JCL Member $31IDMS1
Member $31IDMS1 is generated in the Abend-AID installation sample library (CTL)
to link the Abend-AID for IDMS code with the IDMS module IDMSIDMS.
Notes:
a.
The Abend-AID interface to IDMSIDMS is called by CWAAIDMS and is added by
running $31IDMS1. This interface must be accessed at execution time before
IDMSIDMS. If it is not, Abend-AID issues a message that Abend-AID for IDMS is
not installed. In addition, verify that you are using a copy of IDMSIDMS that
does not already contain the CWAAIDMS CSECT. If it does, $31IDMS1 will fail
with a user abend and store the resulting IDMSIDMS in a library that is accessed
at execution time by the IDMS application program.
b.
If a new release of IDMS is installed or maintenance is applied to IDMSIDMS,
repeat step 3.
c.
No additional steps are required for sites using DBSTATS.
