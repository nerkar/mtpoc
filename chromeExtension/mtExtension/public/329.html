External Security Considerations
Chap 23
The Abend-AID Viewer works with external security packages (RACF, ACF2, and Top
Secret) using the RACROUTE interface to provide secure access to authorized Abend-AID
resources and functions. This chapter describes the types of RACROUTE requests
performed by the Abend-AID Viewer, and provides additional information about using
external security with the Abend-AID Viewer.
The chapter is organized into two sections. The first describes changes you must make to
your site’s security rules or parameters if you are using an external security package. This
section is further divided into subsections that list the security considerations for the
Abend-AID Viewer, batch Abend-AID, and Abend-AID for CICS.
The second section describes additional changes needed to enable RACROUTE calls when
running under either the ACF2 or Top Secret external security packages.
External Security Requirements for All Sites
This section describes required action you must take if your site has an external security
package installed, regardless of whether you explicitly enable the Abend-AID Viewer
external security interface through the specification of external security viewing server
configuration parameters. The section describes granting authority for the following:
•
Default authority for the customization file
•
Defining a user identifier for BDCAS, TDCAS, and viewing server
•
Specifying the user identifier to the BDCAS, TDCAS, and viewing server
Granting Default Authority to the Customization File
You must allow universal READ access to the customization file. This access allows
logged-on Abend-AID Viewer users, CICS regions, the CSS CWDDALLU (formerly,
CWAASDUT and CWFXSDUT) batch file utility, and product facilities such as the MVS
post-dump exit to obtain site-specific information when required. The delete function on
the Server List screen requires UPDATE or greater authority to enable the user to delete
inactive server records from the customization file.
Note:
This step is required if you are using the MVS post-dump exit, and is strongly
suggested even if you are not using this facility. If you do not complete this step,
you must grant READ authority to the customization file to the following:
•
CICS regions
•
All users using TSO/ISPF viewing access
•
All users using the CSS CWDDALLU batch file utility.
Defining a User Identifier for BDCAS, TDCAS, and Viewing Server
If your Abend-AID or user datasets are defined with external security protection, then
you must define a user ID for each batch dump capture address space (BDCAS),
transaction dump capture address space (TDCAS), and viewing server on your system to
your security package. The user ID is used to identify these various address spaces to the
external security package. These address spaces use the user ID to obtain authorization to
access the resources they need.
