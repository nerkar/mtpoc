1. Generate the DB2 Installation JCL
If your siteâ€™s DB2 plan name is different from the displayed plan name, change it on
this screen. You must specify an entry in the subsystem list for each of your site's DB2
subsystems for which you are configuring Abend-AID support. Use the line
commands to I (Insert) new entries and E (Edit), R (Replicate), or D (Delete) existing
entries.
When you enter a line command (except D), the Abend-AID Installation: Site-specific
DB2 Support screen is displayed as shown in Figure 6-4 on page 6-6. Specify each
individual value on the screen, and then issue the GENERATE primary command to
create the JCL to configure and verify Abend-AID's DB2 support on the subsystem.
In the pop-up window that is displayed, specify the name of a partitioned dataset to
receive the generated JCL members. The member name of each JCL job stream to be
created is suffixed with
nxxxx

n
represents the DB2 release and
xxxx
represents
the DB2 subsystem name. A warning pop-up window is displayed if the dataset
currently contains previously generated DB2 installation JCL for the same subsystem.
