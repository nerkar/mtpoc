LE Assembler User Exit
LE allows the definition of an Assembler user exit. It gets invoked at five processing
points:
•
First enclave initialization
•
First enclave termination
•
Nested enclave initialization
•
Nested enclave termination
•
PROCESS termination (LE termination)
Because this exit does not get invoked for unhandled error conditions, it is of no value
for generating transaction dumps, and is not required for Abend-AID for CICS.
Adding Support for New CICS Releases
For Abend-AID for CICS only, this section describes how to install support for additional
releases of CICS. Add support for any release of CICS Transaction Server that is supported
by this release of Abend-AID for CICS and for which your site requires viewing support in
the Abend-AID Viewer.
The Abend-AID for CICS installation sample library (CTL) contains five sample jobs for
each release of CICS, as follows:
•
CxxxRECV:
SMP/E receive the Abend-AID for CICS FMID and preventive service to
support the indicated CICS release.
•
CxxxAPLY:
SMP/E apply the Abend-AID for CICS FMID and preventive service to
support the indicated CICS release.
•
CxxxACPT:
SMP/E accept the Abend-AID for CICS FMID and preventive service to
support the indicated CICS release.
•
CxxxLTRC:
Links the trace formatting module for the indicated CICS release.
•
CxxxSYSD:
Add the DSECT images for the indicated CICS release to the Abend-AID
for CICS system DSECT file.
The xxx in the member names represents the release level of CICS.
The rest of this section provides detailed instructions for adding CICS support.
