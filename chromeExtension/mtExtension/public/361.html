Controlling the Abend-AID Subsystem
App C
This appendix describes starting and stopping the Abend-AID subsystem.
Starting the Abend-AID Subsystem
You can start an Abend-AID subsystem by executing the
first
step in any
one
of the
following installation sample library (INS) members:
•
$68BDCAS
•
$67TDCAS
•
JCLONEKY.
Alternatively, you can update the JCL in member JCLAACNL to start the subsystem.
For
each of these jobs, ensure that the EXEC statement reads as follows:
//START EXEC PGM=FDBASUBS,REGION=1M,PARM='INIT,aasubsystemname'
Note:
This job step starts the Abend-AID subsystem only, not the BDCAS, TDCAS, or
HOTKEY address space.
Stopping the Abend-AID Subsystem
If you need to stop the Abend-AID subsystem, use the sample JCL in installation sample
library (INS) member JCLAACNL. Ensure that the EXEC statement reads as follows:
//START EXEC PGM=FDBASUBS,REGION=1M,PARM='STOP,aasubsystemname'
The
aasubsystemname
variable on the parameter statement must match the name of the
Abend-AID subsystem that you specified in the $68BDCAS, $67TDCAS, or JCLONEKY
member JCL. Ensure that the parameter specifies
STOP
, and not INIT, to stop the
subsystem.
Note:
This job stops the Abend-AID subsystem only, not the BDCAS, TDCAS, or
HOTKEY address space.
