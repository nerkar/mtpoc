Chap 14
Notes:
1.
Compuware recommends that you use the online facility for creating Abend-
AID/Abend-AID for CICS user-defined diagnostics. See “Creating/Maintaining User-
Defined Diagnostics” on page 18-3 for more information.
2.
The Abend-AID for CICS user-defined diagnostics described in this chapter are for
Abend-AID for CICS only. If your site does not plan to install Abend-AID for CICS,
you may skip this chapter.
Besides the online facility for creating user-defined diagnostics, Abend-AID for CICS
includes an additional facility for providing your own diagnostic text modules for
transaction abends. This facility is designed to support transaction abend codes that are
not normally recognized by Abend-AID for CICS, including those for applications
developed in-house and vendor packages used at your site. The text you provide displays
on the Abend-AID for CICS Diagnostic Summary when a transaction dump occurs for
that abend code.
Notes:
1.
The user-defined diagnostic facility supports only transaction abend codes; it does
not support region dumps.
2.
The facility cannot be used to replace diagnostic text for standard abend codes
recognized by Abend-AID for CICS.
Adding New User-Defined Diagnostic Text
Adding user-defined diagnostic text is a two-step process that you must perform for each
text module you want to add. The process involves:
1.
Coding the text module.
2.
Assembling, link-editing, loading the text module into the Abend-AID Viewer
customization file.
These steps are described in detail in the following sections.
1. Code the Text Module
Installation sample library (CTL) member DIAGSAMP contains the assembler macro
definition used to create a user-defined diagnostic text module. The macro creates a
complete diagnostic text module, including the necessary CSECT and END cards.
The macro statement at the bottom of the macro definition contains a sample diagnostic
text. Modify the sample as follows:
•
Ensure that the CODE parameter reflects the transaction abend code for which you
are customizing the diagnostic text.
•
Code the text of the diagnostic, containing 1—n text lines, with the following
restrictions:
–
Each line must begin in column 16.
