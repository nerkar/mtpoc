Stopping the Viewing Server, BDCAS, or TDCAS
You can use three different commands to stop a viewing server, BDCAS, or TDCAS. With
each method, the viewing server, BDCAS, or TDCAS waits at least 30 seconds before it
terminates to allow current users to exit and for the transaction load to be closed down.
Viewing servers, BDCASs, and TDCASs can be stopped using the following methods:
•
Abend-AID Viewer SHUTDOWN command
•
MVS STOP command
•
MVS MODIFY command (refer to “Stopping the Viewing Server, BDCAS, or TDCAS”
on page A-3).
Abend-AID Viewer SHUTDOWN Command
To stop the viewing server, BDCAS, or TDCAS from within the Abend-AID Viewer, type
SHUT
in the COMMAND (or OPTION) field on any Abend-AID Viewer screen and press
Enter. If the viewing server, BDCAS, or TDCAS was started as a job, you will get either
message IEF355A or IEF352I.
Note:
For viewing servers only, you can restrict access to this command by coding the
SHUTDOWN subparameter of the EXTERNAL_SECURITY_FUNCTION_CHECK
viewing server configuration parameter. Refer to “Parameters to Enable External
Security for Viewing Servers” on page 15-8 for more information.
nnnnn
Timer value that specifies the number of seconds to elapse before the viewing server
or TDCAS stops. The default is 30 seconds.
IMMED
Timer value that specifies that the viewing server or TDCAS is to stop immediately.
MVS STOP Command
To stop the Abend-AID Viewer from the MVS console, type
P
jobname
where
jobname
is
the job name of the viewing server, BDCAS, or TDCAS.
