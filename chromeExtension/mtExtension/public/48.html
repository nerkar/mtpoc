1. Allocate the Abend-AID Customization Load Library ($60CUSTL)
If you’re installing batch Abend-AID, use the JCL in Installation sample library (CTL)
member $60CUSTL to allocate the new Release 16.05 customization load library you
specified in Figure 3-6 on page 3-4.
This customization load library is not managed by SMP/E. This library must exist before
you can continue on to Chapter 4, “Batch Abend-AID Configuration”. The load modules
that will be contained in this library must be available when the Abend-AID hook is
started as well as at Abend-AID execution.
2. Allocate and Load the System DSECT File ($61DSECT)
The system DSECT file is a VSAM RRDS that contains the data required to format DSECT
displays in the Abend-AID Viewer. The file is allocated and formatted using the
Compuware Shared Services (CSS) utility CWDDALLU, and accessed using the CSS DDIO
access method. Installation sample library (CTL) member $61DSECT contains a
procedure to allocate the system DSECT file.
$61DSECT is tailored based on the information you specified in the installation dialog.
Before you submit $61DSECT, verify that the dataset name and unit and volser or SMS
storage and management class information is correct.
Notes:
1.
If this job ends with RC=04, verify that the SMP/E ACCEPT job, described in the
Compuware Installer Mainframe Products SMP/E Installation Guide
, completed successfully.
2.
You may need to increase the size of the DSECT file when adding support for an
additional CICS Transaction Support version.
3. Define the MVS DSECT Images ($61MVSD)
Installation sample library (CTL) member $61MVSD contains the procedure that executes
the Assembler language processor to assemble and post-process the MVS DSECT images
and load the output into the system DSECT file. $61MVSD is tailored based on the
information you specified in the installation dialog.
Before executing the $61MVSD job, verify the following:
•
The CSSL variable specifies the load library that contains the CSS Assembler language
processor.
•
The DSCT variable specifies the system DSECT file into which the MVS DSECT images
are to be stored. The system DSECT file is created by installation job $61DSECT.
4. Allocate, Copy and Upgrade, or Upgrade in Place a Customization File ($62CUST)
Review and then execute installation sample library (CTL) member $62CUST. This
member contains a tailored job to either allocate, copy and upgrade, or upgrade in place
the Abend-AID Viewer customization file and to load base default records based on the
customization file information you entered in Figure 3-6 on page 3-4.
Notes:
1.
The customization file is defined with share options (4 3). This allows multiple
Abend-AID viewing servers running in a single MVS image UPDATE access to the
customization data. Abend-AID programmatically ensures WRITE integrity for the
customization file.
