Copying Module MSDMCS1D to an Authorized Library
Note:
A copy of MSDMCS1D is required on every MVS image on which you intend to
use the remote notification function, but where the Abend-AID Viewer
authorized load library (SKAZAUTH) is
not
accessible.
It is not required on systems where the Abend-AID Viewer authorized load library
(SKAZAUTH)
is
accessible, because MSDMCS1D resides in the Abend-AID Viewer
authorized library.
When you configure your action definitions, as described in “Creating Action
Definitions” on page 21-1, you must create a job card for each MVS SYSID on which
either dumps can be taken or users notified of those dumps. A //JOBLIB DD card must be
included for the authorized library containing module MSDMCS1D on that MVS system,
unless the module is copied to an authorized system linklist library.
Creating Batch Dump Contact Information Entries
Note:
This option is available only if batch Abend-AID is installed on this viewing
server.
Batch dump contact information entries specify which contact information is associated
to which batch dumps.
To create batch dump contact information entries, complete the following procedure:
1.
Display the Batch Dump Contact Information screen, shown in Figure 21-10, using
one of two methods:
–
Tab to the Batch Dump Contact Information Definitions option on the Action
Definition menu (Figure 21-2 on page 21-2), and then press Enter.
–
Type
ACTBATCH
or
=3.4
in the COMMAND (or OPTION) field on any
customization
screen and press Enter. You cannot use these fast-path commands
unless you have accessed the customization facility.

Batch Dump Contact Information Screen

information entries that specify which contact information is associated to which
batch dumps. This screen initially displays the wildcard character (*) in the Job
Name, MVS SYSID, Program, and Abend Code fields. These fields are used to qualify
the dumps for which you want notification.
Use the I or R line command to insert a new entry or replicate an existing entry as a
template.
