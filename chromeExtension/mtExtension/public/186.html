XE_HTTPS= NO|yes
Optional for the Abend-AID Web Browser Interface only. When the Xpediter/Topaz
Workbench interface is used, specifies whether the URL for the Abend-AID report is
created with the HTTP or HTTPS prefix.
If the URL is created with the HTTPS prefix, the TCP/IP port that is defined to the
Abend-AID server must be a secured port using AT-TLS. The default is
NO.
Parameters to Enable External Security for Viewing Servers
The parameters listed below control when Abend-AID makes external security calls to
verify user access.
The viewing server itself runs using the authority of the individual who starts the
viewing server address space. Chapter 23, “External Security Considerations” provides
more information about how Abend-AID interfaces to external security managers, and
the authority you must grant to the viewing server address space.
Compuware strongly recommends you review the information in that chapter before
specifying any external security viewing server configuration parameters to ensure
you have a complete understanding of how Abend-AID interfaces to external
security packages.
EXTERNAL_SECURITY_ENABLED=NO|yes
Specifies whether Abend-AID makes calls to external security packages. You must
specify
YES
for this parameter for any other EXTERNAL_SECURITY_* parameter to be
valid. The default value for this parameter is
NO
.
NO
Abend-AID external security is not enabled, and the following is in effect:
•
No external security calls are issued.
•
Users are required to specify only a user ID when they access Abend-AID. A
password is not required.
•
Access to region dump datasets, report databases, and source listing files is
governed by the authority of the user ID that submitted the viewing server.
