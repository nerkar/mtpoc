Configuring Abend-AID for CICS DB2 Support
Notes:
1.
If you were not licensed for the option when you initially installed Abend-AID for
CICS but subsequently became licensed, you can add the support it provides by
completing the procedure described below.
2.
If you are migrating to this release of Abend-AID for CICS from an earlier release,
review the information in “Concurrent DB2 Releases” on page 6-13 about migrating
the Abend-AID for CICS DB2 option.
This section describes the procedure to configure the Abend-AID for CICS DB2 option. If
you are licensed for this option, perform the steps described in this section.
Abend-AID for CICS DB2 Support Security
Abend-AID for CICS DB2 support uses the Recoverable Resource Manager Services
Attachment Facility (RRSAF) to connect to DB2 in the transaction dump capture address
space (TDCAS). Review your security procedures for connecting an address space to DB2
using RRSAF. You may need to define a profile for the DB2 resource class to permit
Abend-AID for CICS DB2 access or give universal access to its profile. Refer to IBM’s
DB2
Administration Guide
for examples on enabling RACF checking for the DSNR class or
consult your security administrator.
For additional DB2 security requirements, see “Granting Authority for the TDCAS” on
page 23-14. If your site uses ACF2 or Top Security as your security package, also review
“Enabling RACROUTE under ACF2 and Top Secret” on page 23-15 for required SAFDEF
information.
