Abend-AID Installation and Customization Guide
The following syntax is valid for the MODIFY command:
jobname
The transaction dump capture address space job name.
STatus
Displays whether the DFHRPL_READ function is enabled or disabled. This parameter
also displays a list of the dataset entries with their statuses: open, excluded, disabled.
ENable
Turns on the DFHRPL_READ function. This parameter does not change the status of
any excluded datasets.
DIsable
Turns off the DFHRPL_READ function. This parameter also goes through the dataset
list. For each dataset entry, it closes the dataset if it is open, and then flags the
dataset entry as being deleted. The only exception to this process are the excluded
datasets entries, which are ignored.
EXclude, dataset
Excludes the specified dataset from being processed in the TDCAS. The program goes
through the dataset list entries searching for a name match. If it finds one, and if the
dataset is open, it closes the dataset and then flags the entry as being excluded. If it
does not find a name match, it creates a new entry and flags it as being excluded.
Masking of the dataset name is supported. While processing the list, all datasets that
match the mask are closed and flagged as deleted. A new entry is then created with
the masked dataset name.
Once a dataset is excluded, it remains excluded until either an INCLUDE command is
issued or the server is shut down.
INclude, dataset
Includes a dataset that was previously
excluded
to be processed in the TDCAS. The
program goes through the dataset list entries searching for a name match. If it finds
one, and the dataset is excluded, it flags the entry as being deleted. Then when the
TDCAS is processing a dump and it needs information from a CSECT that is in that
load library, that dataset will be allocated and opened in the TDCAS. If it doesn't find
a matching name or the dataset isn't excluded, an error message is issued.
