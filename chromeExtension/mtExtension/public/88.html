This step describes Abend-AID for CICS’s DB2 package versioning, how Abend-AID
for CICS uses the DB2 system catalog tables, binding the Abend-AID for CICS DB2
DBRMs, and granting privileges for the plan.
Abend-AID for CICS DB2 Package Versioning
Abend-AID for CICS DB2 support now uses package versioning in its precompile
preparation process. To take advantage of this method, the Abend-AID for CICS
DBRMs must be bound into packages at installation time. When maintenance is
applied, use installation sample library (CTL) member $22
nxxxx
, where
n
represents
the one-numeric character DB2 release identifier and
xxxx
is the four-alphanumeric
character DB2 subsystem ID. The replacement DBRMs are bound using the same
collection ID and package names as those used during the installation, allowing DB2
to locate the needed package during processing. This method also eliminates the
need to rebind the plan after the latest packages are bound.
How Abend-AID for CICS Uses the DB2 System Catalog Tables
The Abend-AID for CICS DB2 option uses SQL statements to obtain information from
the DB2 system catalog tables. This requires that a DB2 bind be performed to enable
the SQL statements. If you do not bind the DBRMs, the Abend-AID for CICS DB2
option still functions, but the DB2 catalog tables are not accessed and information is
missing that is normally provided by Abend-AID for CICS.
In order to obtain the information from the DB2 catalog tables, Abend-AID for CICS
connects to a DB2 subsystem. A health check is available to provide information
about the Abend-AID for CICS DB2 connections. See Appendix H, “Abend-AID
Health Checker” for more detailed information.
If the DBRMs are bound, the following DB2 system catalog tables are accessed:
–
SYSIBM.SYSCOLUMNS
–
SYSIBM.SYSDBRM
–
SYSIBM.SYSFIELDS
