Before starting URLPUB support initialization, CWASACTL must be loaded into the LPA
or it must reside in an APF-authorized library on the link list. To dynamically add
CWASACTL to the LPA, issue the following MVS command substituting the installed

Enabling URLPUB Processing
To start the HOTKEY/URLPUB address space and/or enable URLPUB processing:
1.
Execute sample JCL in installation sample library (CTL) member JCLURLPB.
2.
Ensure that the UPPARM is set to 'REQ=START.'
Note:
URLPUB processing requires the use of an Abend-AID subsystem. The first step of
the JCLURLPB job starts the Abend-AID subsystem. Only one Abend-AID
subsystem is required for each MVS image (LPAR). Therefore, if an Abend-AID
subsystem is active in your environment, you do not need to run this step.
Review the comments in JCLURLPB regarding the processing options and subparameters
for implementing the URLPUB support at your site. Add a //FDBDCUST DD statement for
each customization file to be considered for Publish URL support.
If problems occur, URLPUB support initialization can be resubmitted via JCLURLPB as
issues are resolved. No special processing is necessary for a resubmit.
Stopping URLPUB Processing
If you need to stop URLPUB processing only:
1.
Execute only the
second
step of sample JCL in installation sample library (CTL)
member JCLASCNL.
2.
Ensure that the UPPARM is set to 'REQ=STOP'.
If you need to stop both the HOTKEY and URLPUB functions and bring down the
HOTKEY/URLPUB address space:
1.
Execute
both
steps of sample JCL in JCLASCNL.
2.
Ensure that the OKPARM is set to 'REQ=STOP'.
3.
Ensure that the UPPARM is set to 'REQ=STOP'.
Note:
This job stops the HOTKEY/URLPUB address space only. See Appendix C,
“Controlling the Abend-AID Subsystem” for more information about the Abend-
AID subsystem.
Installing VS FORTRAN Support
Abend-AID helps FORTRAN programmers diagnose application program errors by
reporting information that VS FORTRAN’s normal error recovery process obscures. VS
FORTRAN run-time routines intercept application program abends when its SPIE and
STAE run-time options are in effect. VS FORTRAN then terminates the application with a
U0240 abend, hiding the original error. Although basic information about the
application’s problem may be provided by VS FORTRAN messages, this information is
usually insufficient to help resolve errors. When Abend-AID VS FORTRAN support is
installed, however, Abend-AID provides crucial diagnostic information pertaining to the
original error.
