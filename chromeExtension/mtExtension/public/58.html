6. Verify Batch Abend-AID External Security Considerations
If your site uses an external security package, updates may be required in order to allow
the batch Abend-AID to execute successfully. See “External Security Requirements for All
Sites” on page 23-1 and “Batch Abend-AID Security Requirements” on page 23-13 for
more information. If your site uses ACF2 or Top Secret as your security package, also
review “Enabling RACROUTE under ACF2 and Top Secret” on page 23-15 for required
SAFDEF information.
7. Start the Batch Dump Capture Address Space
The Abend-AID batch dump capture address space (BDCAS) is required for all sites
installing batch Abend-AID. The BDCAS must be active when Abend-AID is active. If the
BDCAS is not active, you will not be able to generate an Abend-AID report, even to
SYSOUT.
To start the batch dump capture address space (BDCAS), submit the Abend-AID
installation library (CTL) member $68BDCAS job that was generated. See Appendix A,
“Controlling the Abend-AID Viewing Server, BDCAS, and TDCAS” for more information
about controlling the BDCAS.

Execute the CWINSTAL Utility to Install the Abend-AID SVC51 Interface
Note:
The BDCAS should be active before starting the Abend-AID SVC51 interface.
Ensure that you complete the previous step before proceeding with this step, or
you will receive the following error message.
ABINS140 RETURN CODE FROM CWBCSTAT = xxxxxxxx. BDCAS IS NOT AVAILABLE
This message is a reminder that without an active BDCAS, no Abend-AID reports
will be generated. Starting a BDCAS after receiving this message will allow
Abend-AID to successfully generate reports.
Submit Abend-AID installation library (CTL) JCL member $12INSTL to start batch Abend-
AID for testing. The CWINSTAL utility starts and stops the batch Abend-AID SVC51
interface by specifying parameters using the PARM field of the execution statement. After
the Abend-AID SVC51 interface is installed, it automatically begins intercepting calls to
SVC51 when an application program failure occurs.
