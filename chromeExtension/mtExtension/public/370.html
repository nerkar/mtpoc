Configuring Abend-AID on Multiple MVS Images
This section documents two basic configurations supporting Abend-AID on multiple MVS
images — both in a coupled multiprocessor and parallel sysplex configuration:
•
Multiple viewing servers sharing a single customization file across MVS images.
•
One viewing server supporting multiple MVS images.
In both of these configurations, shared DASD is a requirement. In addition, you need to
use Global Resource Serialization (GRS) or an equivalent product to protect the integrity
of Abend-AID files shared across MVS images.
See “GRS Considerations” on page 15-26
for more information.
Note:
Abend-AID does not use the MVS coupling facility for data sharing and locking in
a parallel sysplex. However, Abend-AID uses shared DASD and GRS to provide
data sharing and locking.
Multiple Viewing Servers Sharing Files Across MVS Images
This section provides a configuration example of two viewing servers on each of two MVS
images. It first describes the datasets used by the viewing servers, and then provides an
example of viewing access configuration. The information in this section is useful for
sites that have multiple MVS images, and want to share a single customization file
between images. If you have multiple MVS images but want to define one customization
file per MVS image, see the example provided in “Two Viewing Servers and One TDCAS
for a Single MVS Image” on page D-3 or “Two Viewing Servers and Two TDCASs” on page
D-6.
