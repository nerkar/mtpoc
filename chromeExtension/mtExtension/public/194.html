ARM_ENABLED=NO|yes
Specifies whether the server is to be registered for restart with ARM. The default is
NO
.
ARM_ELEMENT_SUFFIX=servername|suffix
Specifies the one- to eight-character suffix of the ARM element name used to register
with ARM.
The element name used by the Abend-AID Viewer to register with ARM is
KMPssidsuffix, where
KMP
is a constant,
ssid
is the four-character MVS system ID, and
suffix
is the ARM_ELEMENT_SUFFIX.
The element name is used by ARM to select a
restart policy. Each Abend-AID viewing server using ARM requires a unique suffix.
Unless you specify a different suffix, the viewing server uses the default, the server
name parameter value, which is the one- to eight-character server name specified in
the server execute statement.
Note:
If the automatic restart manager (ARM) restarts a job, JES discards all non-spin
SYSOUT datasets created during the previous execution. SPIN=UNALLOC is
automatically included on the //FDBDLOG DD statement for the SYSOUT
datasets in the viewing server generated JCL to avoid losing valuable debugging
information. For example:
//FDBDLOG DD SYSOUT=*,SPIN=UNALLOC
Modifying the Viewing Server JCL ($65SRVR)
Installation sample library (CTL) member $65SRVR contains sample startup JCL for the
viewing server. Review and modify the sample JCL according to the guidelines in this
section.
JOB or PROC Statements
You can run the viewing server as either a job or a started task. If you want to run the
viewing server as a started task, change the JOB statements to PROC statements, and
modify the viewing server JCL according to the instructions in the following sections.
Then, copy the PROC to a library in your system PROCLIB concatenation.
If you run the viewing server as a job, modify the viewing server JCL according to the
instructions in the following sections. Then, submit the viewing server JCL when you
want to start the viewing server.
Notes:
1.
The viewing server JCL is always generated with a TIME=1440 parameter on the
execute statement to prevent the viewing server from timing out, regardless of the
amount of CPU time it uses. If necessary, modify this parameter to your site’s
requirements.
2.
The viewing server region size defined on the execute statement is 34 megabytes.
This is the minimum recommended region size for the viewing server.
Dump Information File Reorganization
The first step in the viewing server JCL performs a reorganization of the dump
information file. Although including this step is optional, Compuware strongly
recommends you include it to help reduce the impact of the large scale fragmentation to
which this file is often subjected. If the dump information file fragments to the point
where new dump information cannot be added, the viewing server fails and the file must
be manually reorganized.
