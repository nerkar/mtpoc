Modifying CICS Transaction Dump Handler Profiles
Note:
This option is for use with Abend-AID for CICS only. Add the $DEFAULT dump
handler profile entry to any existing region configuration entries if you want to
use this feature. See “Tran Dump Handlers” on page 20-9 for more information.
If you have Abend-AID for CICS installed, on the Region Configuration screen (option 1
on the Dump Capture and Processing Menu), you can assign regions (using job name,
APPLID, and MVS SYSID as qualifiers) to use the $DEFAULT transaction dump handler
profile or profile(s) you create. Multiple regions can use the same profile but if you don’t
specify a profile, none is used. You can update the Compuware-supplied default profile,
but you cannot delete it.

Dump Capture Profiles Screen for Transaction Handlers
The Dump Capture Profiles screen for transaction handlers lists the transaction
dump handler profiles. When you initially access the screen, the $DEFAULT handler
profile is displayed. This member contains the default profile values supplied by
Compuware, and can be used as a template for profiles you create. If the Compuware
defaults are acceptable, you don’t have to do anything, and Abend-AID for CICS uses
the default dump handler profile for region configuration entries that specify
$DEFAULT in the Tran Dump Handlers field on the Region Configuration screen
(Figure 20-5 on page 20-8). The Dump Capture Profiles screen for transaction dump
handlers supports a maximum of 2,031 entries.
