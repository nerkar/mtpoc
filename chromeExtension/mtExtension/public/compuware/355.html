Closing and Reopening the FDBDLOG File
The MVS MODIFY command allows you to close and reopen the viewing server, BDCAS,
or TDCAS FDBDLOG file.
The next time you issue the LOGSPOOL command,
you can
print, delete, or offload the closed file. Executing this command can reduce the amount
of spool space consumed by the viewing server, BDCAS, or TDCAS in continuous
operation environments.
You can also execute the MVS MODIFY command through an
automated operations facility.
Note:
To free the initial FDBDLOG file, stop the viewing server, BDCAS, or TDCAS.
Modify the FDBDLOG DD statement to indicate a specific SYSOUT class other
than * and to include the FREE=CLOSE parameter. Restart the viewing server,
BDCAS, or TDCAS.
The following syntax is valid for the MODIFY command:
jobname
jobname
The viewing server, BDCAS, or TDCAS job name
Controlling Opening and Closing of Load Libraries
The MVS MODIFY command lets you dynamically open and close the CICS load library
for an abending module for the purpose of reading in CSECT information during dump
processing in the TDCAS. Using the DFHRPL keyword, this command also lets you
dynamically open and close individual TDCAS datasets or groups of datasets. Although
this CSECT support greatly increases performance during dump processing, there are
times when the function may need to be disabled or some of the load libraries closed,
such as for dataset backups.
This command allows you to temporarily override the TDCAS DFHRPL_READ parameter
if you are using the default setting,
YES
, or to provide DFHRPL_READ support on a
temporary basis if your site has set the parameter to NO. See “DFHRPL_READ=YES|no” on
page 12-5 for more information.
