Batch Abend-AID Security Requirements
Resource Security
Batch Abend-AID reads the load libraries associated with the abending program. If users
or programs are denied read access to application program load libraries, dataset security
violations may be generated by security applications such as RACF or ACF2. To prevent
these violations, the #XAAXIO load module requires read access to the application
program load library.
Report Shared Directory and Database Files
All user IDs, under which jobs are submitted, require UPDATE authority to the report
shared directory and all report databases attached to that shared directory in order to
write the Abend-AID report. See “Controlling User Access to Abend-AID Datasets” on
page 23-7 for more information.
Granting Authority for the BDCAS
If your site has batch Abend-AID installed on this server, ensure that the owner of the
batch dump capture address space (BDCAS) has the level of authority listed for the
Abend-AID files shown in Table 23-4.
Granting Authority for the BDCAS to RRSAF
Abend-AID for DB2 processing uses Recoverable Resource Manager Services Attachment
Facility (RRSAF) to access the DB2 catalog.
Therefore, the BDCAS must have access to
DSNRLI, which is the RRSAF language interface load module. If your site uses an external
security package that restricts access to system modules, you may need to write a rule
authorizing the BDCAS to access DB2 DSNR* resources.
Specifying the External_Security_Enabled parameter
Users are required to log onto the BDCAS using a valid user ID and password if you
specify
YES
for the EXTERNAL_SECURITY_ENABLED BDCAS configuration parameter, as
described in “EXTERNAL_SECURITY_ENABLED=NO|yes” on page 8-3.
The function value for the logon function is
LOGON.TC
.
You must write the appropriate rules for your external security package, based on the
following resource name batch Abend-AID generates for the logon function:
prefix.SERVER.LOGON.TC.servername
The minimum authority you must grant users for this function is READ.
The user ID that starts the viewing server must also have this authority.
