LE Support Considerations
This section describes using Abend-AID for CICS with transactions running under IBM’s
Language Environment (LE). It also provides some installation information and answers
to commonly asked questions. Because this chapter does not duplicate information
provided by IBM, refer to the appropriate IBM documentation for more detailed
information about LE.
Abend-AID for CICS provides LE support for COBOL and PL/I programs. This support
includes all supported versions of COBOL and PL/I. In addition, selected LE control
blocks and areas are shown on the Control Blocks/Storage screen, and the LE Options
Control Block (OCB) is formatted.
Activating Abend-AID for CICS under LE
No special installation instructions are required to use Abend-AID for CICS with
Language Environment for supported COBOL and PL/I releases. Once your environment
has been set up to allow CICS to take transaction dumps, simply activate Abend-AID for
CICS using the AAON transaction.
If you have problems getting Abend-AID for CICS to take transaction dumps, first
confirm that the environment is correctly set up. Turn off Abend-AID for CICS using
AAON OFF and check that IBM CICS transaction dumps are correctly taken. Once you
verify that the environment is correct, you can then activate Abend-AID for CICS using
AAON ON. See Appendix B, “Supplied Abend-AID for CICS Transactions” for more
information about using this transaction.
Abends occurring for CICS transactions under LE cause a transaction dump to be taken
only
for CICS-detected errors — for example, ASRA and AEIx. To cause a transaction dump
for COBOL software-raised conditions (1nnn) and errors of severity level 2 and greater for
Enterprise PL/I, see “Transaction Dumps Occurring under LE” on page 13-3. For example,
in the new releases of PL/I, a ZERODIVIDE is now an ERROR condition.
