Migration Configuration Procedures
These configuration procedures are intended to reduce configuration time by maximizing
the use of components from a currently installed release of Abend-AID. These procedures
assume the installer is familiar with the Abend-AID configuration process. For additional
details on these configuration steps, see the following chapters in this guide:
•
Chapter 3, “Abend-AID Viewer Configuration”
•
Chapter 4, “Batch Abend-AID Configuration”
•
Chapter 5, “Abend-AID for CICS Configuration”
•
Chapter 6, “Database Support Configuration”
Notes:
1.
You must be migrating from Abend-AID Release 11.2 or a more current version.
2.
You must not have updated the DB2, CICS, or z/OS release since the prior version of
Abend-AID was updated with release updates for DB2, CICS, or z/OS.
3.
Depending on how you received the Abend-AID product files, you must have already
executed the SMP/E installation jobs as described in the
Compuware Installer Mainframe
Products SMP/E Installation Guide
before proceeding to the configuration process.
Migrating the Viewing Server
Follow these steps to update the current viewing server to Abend-AID Release 16.05:
1.
Generate the Abend-AID Viewer Release 16.05 configuration and customization JCL
by following the procedure described in Chapter 3, “Abend-AID Viewer
Configuration” that uses your input from the Configure Abend-AID Viewer option.
Enter the
generate
command on the last dialog panel.
2.
Only the following subset of Abend-AID Viewer Release 16.05 configuration and
customization jobs needs to be completed when reusing files from a previous Abend-
AID release.
–
$60CUSTL: To create the new Abend-AID customization load library (batch
Abend-AID only).
–
$62CUST: To create new Abend-AID Viewer customization file or to upgrade the
current customization file to Abend-AID Release 16.05 format.
–
$65SCPY: To create and populate the new FDBDCLST dataset.
