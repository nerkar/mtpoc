3. Verify the Abend-AID Datasets
The following DD statements are generated with dataset names that are created from the
values specified on the installation dialog. Ensure they are correct:
//STEPLIB DD DISP=SHR,DSN=compware.lcx160.SLCXAUTH
// DD DISP=SHR,DSN=compware.paa160.SPAAAUTH
//FDBDRPL DD DISP=SHR,DSN=compware.kaz160.SLCXLOAD
// DD DISP=SHR,DSN=compware.paa160.SPAALOAD
//FDBDCUST DD DISP=SHR,DSN=aavwr.cust
//FDBDPARM DD DISP=SHR,DSN=compware.CTL($68BDPRM)
The //STEPLIB DD points to the authorized step libraries for:
•
Enterprise Common Components
•
Batch Abend-AID
The //FDBDRPL DD points to the load libraries for:
•
Enterprise Common Components
•
Batch Abend-AID
The //FDBDCUST DD points to the Abend-AID Viewer customization file, that is
described in “4. Allocate, Copy and Upgrade, or Upgrade in Place a Customization File
($62CUST)” on page 3-6.
The //FDBDPARM DD points to the BDCAS parameters dataset ($68BDPRM).

5. Verify the BDCAS Log Destination
The //FDBDLOG DD statement specifies a SYSOUT class that is used to record BDCAS
informational and error messages. Compuware recommends that you check this output if
you experience problems using batch Abend-AID, since many BDCAS-related messages
are written there.
The default value is
SYSOUT=*, SPIN=UNALLOC
Note:
SPIN=UNALLOC is
required
to prevent the loss of JES output with the
ARM_ENABLED=YES parameter setting in the viewing server/BDCAS. However,
this default setting is also valid for sites not using the ARM restart option.
6. Verify ABNLIGNR DD Statements
The BDCAS JCL contains an //ABNLIGNR DD DUMMY statement so that batch Abend-
AID does not process BDCAS dumps. These statements should not be removed from the
BDCAS JCL.
