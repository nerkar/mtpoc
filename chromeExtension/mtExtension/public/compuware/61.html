Batch Abend-AID Configuration
4-9
Testing Abend-AID for WebSphere MQ Support
Execute JCLMQ1 and JCLMQ2
If your site is licensed for optional Abend-AID for WebSphere MQ support, you can test
WebSphere MQ support with batch Abend-AID by executing installation library (CTL)
members JCLMQ1 and JCLMQ2. These IVPs test batch Abend-AID’s WebSphere MQ
support with a COBOL application only and do not represent real-world examples. For
each IVP, verify the job card, dataset names, the WebSphere MQ subsystem name, and
the model queue name before submitting the job.
Moving Batch Abend-AID into Production
In this step you stop the test mode of Abend-AID’s SVC51 interface and restart the
interface in production mode, then make final operating preparations.
Note:
Production mode diagnoses failures in all jobs. Test mode diagnoses failures in
specified jobs.
CAUTION:
The SVC51 interface can be run in production and test modes simultaneously. You
can start the test mode without stopping the production mode. You cannot start the
production mode if the test mode is active. If the interface is started twice without a
PARM setting, the first will be production mode and the second will fail because
production mode is already active.
1. Stop the Test Abend-AID SVC51 Interface
Remove the method used for testing Abend-AID:
–
If the new Abend-AID SVC51 interface is started with test job names or job name
masks, stop the test Abend-AID SVC51 interface. Sample JCL is in member
$12INSTL in the Abend-AID installation library. Submit job $12INSTL with:
•
PARM=‘STOP,*’
•
//STEPLIB and //ABENDLIB DD statements that point to the authorized
library(ies) containing the new CWINSTAL and ABENDAID load modules
respectively.
–
If Abend-AID was installed using the global option, “Disable Abend-AID
processing” on page 20-44 set to
YES
, then change this to
NO
. Abend-AID is then
in effect without adding an //ABNLENAB DD statement to the JCL. See “Disable
Abend-AID processing” on page 20-44 for more information.
2. Verify SVC51 Interface for Reinstallation
If you are installing the Abend-AID SVC51 interface for the first time, skip to step “3.
Make the New Abend-AID Load Modules Accessible for All Jobs” on page 4-10. For a
reinstallation, to verify that the interface was installed dynamically, submit job
$12INSTL with PARM=‘LIST’.
If the old interface was installed dynamically, submit job $12INSTL with:
–
PARM=‘STOP’
–
//STEPLIB and //ABENDLIB DD statements that point to the authorized
library(ies) containing the new CWINSTAL and ABENDAID load modules
respectively.
