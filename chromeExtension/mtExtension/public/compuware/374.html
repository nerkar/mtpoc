Dataset Usage of One Viewing Server Supporting Two MVS Images
Notes:
1.
A single viewing server, SERVER, is running on MVS2. CICS regions CICSTST1 and
CICSTST2 are also running on MVS2. CICS region CICSPROD is running on MVS1.
SERVER is supporting all 3 CICS regions, even though it is not running on the same
MVS image as CICSPROD.
2.
In addition to the single viewing server on MVS2, another viewing server can be
created on the other LPAR, MVS1, so that reports can still be viewed if MVS2 suffers
an outage. The MVS1 viewing server can share with the MVS2 viewing server the
customization file, system DSECT file, report shared directory and databases, and the
Scratchpad listing file (CWPDDIO). The MVS1 viewing server needs its own copy of
the other viewing server datasets. Both viewing servers should specify the
SHARED_DIRECTORY_OPTIONS=SHARE viewing server parameter. See
“SHARED_DIRECTORY_OPTIONS=UNSHARE|share” on page 15-13 for more
information.
3.

Transaction databases. Compuware recommends that you allocate at least two
transaction databases per viewing server.
–
Dump information file.
Ensure that these files are part of a GRS ring to protect their integrity. See “GRS
Considerations” on page 15-26 for more information.

are “viewing server-owned” datasets, they must be accessible to the CICS region
running on MVS1. Therefore, they must be allocated on shared DASD.
