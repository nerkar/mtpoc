Transaction Dump Capture Address Space Customization
12-11
The dispatching priority for the TDCAS must be at least as high as, and preferably
higher than, any CICS region connected to it. Failure to do this may result in loss of
critical abend data. If the TDCAS dispatching priority is set lower than that of the
CICS regions, the TDCAS may not get the processing resources it needs to process
abend requests. Inadequate processing resources may cause the abending
transactions to back up in the CICS region until they begin to time out causing the
CICS region to thrash because of the number of active CICS tasks. Often regions will
reach maximum tasks or become short on storage.
Because the time-out value is relatively long in terms of normal CICS task life, the
system can never catch up, ultimately preventing CICS from servicing user requests.
All of this results in still fewer system resources being available for the TDCAS to
process dumps.
In a system that uses MVS workload manager, the TDCAS must be in a service class
that is always guaranteed to get a higher dispatching priority than the CICS regions
that it supports. Typically, this means that the TDCAS should be run as a member of
the SYSSTC class because of the nature of WLM. If the TDCAS is put in the same
group with the CICS regions, then as abend activity in one or several of the CICS
regions goes up, the TDCAS’ absolute dispatching priority is adjusted downward by
WLM (since it consumes more resources to process the higher dump activity).
Conversely, as more CICS tasks sit in abend-wait, the CICS region’s absolute
dispatching priority tends to go up because it is doing less (most of its tasks are
sitting in abend-wait).
There is no danger in setting the TDCAS region to a too high priority. The TDCAS
only consumes resources to process the dump requests made of it. When no dump
requests are active, it consumes very minimal CPU or I/O resources.

Terminals in Abend Wait
If your users complain that their terminals lock up in an ABEND WAIT condition, reduce
the following transaction dump capture global options:
•
ABTIME
•
DBTIME
Reducing these values will increase the number of unprocessed transaction dumps (an
IBM dump is taken for those dumps not processed by Abend-AID for CICS), but doing so
will free the users’ terminals faster.
Note that you must turn Abend-AID for CICS off and back on in the CICS region after
you change a transaction dump capture global option.
