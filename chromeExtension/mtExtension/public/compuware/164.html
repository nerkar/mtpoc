2. Verify the Subsystem and TDCAS Names
The TDCAS JCL has two execute statements: one for the Abend-AID subsystem and one
for the transaction dump capture address space:
•
The first execute statement contains a parameter value that specifies the name of the
Abend-AID subsystem. This value is a four-character name. The default is
CWAA
. The
parameter is specified PARM=’INIT,aasubsystemname’. The INIT must be included in
the parameter. You can specify the same subsystem name for every TDCAS on a single
MVS image.
•
The second execute statement contains a parameter value that specifies the name of
the transaction dump capture address space. This value is a one- to eight-character
name that is used to identify the TDCAS to Abend-AID for CICS components.
Therefore, you must specify a
unique
value for the parameter for each TDCAS you
create on this MVS image.
CAUTION:
The viewing subsystem is not the same as the Abend-AID subsystem. You cannot
share the subsystem between a viewing server and a transaction dump capture
address space (TDCAS). See “Parameter Required to Support TSO/ISPF or CICS
Viewing Access” on page 15-4 for information about specifying the viewing
subsystem name.
3. Verify the Abend-AID for CICS Datasets
The following DD statements are generated with dataset names that are created from the
values specified on the installation dialog. Ensure they are correct:
//STEPLIB DD DISP=SHR,DSN=compware.kaz160.SLCXAUTH
// DD DISP=SHR,DSN=compware.lcx160.SKAZAUTH
// DD DISP=SHR,DSN=compware.kfx161.SKFXAUTH
//FDBDRPL DD DISP=SHR,DSN=compware.kaz160.SLCXLOAD
// DD DISP=SHR,DSN=compware.lcx160.SKAZLOAD
// DD DISP=SHR,DSN=compware.kfx161.SKFXLOAD
//FDBDCUST DD DISP=SHR,DSN=aavwr.cust
//FDBDPARM DD DISP=SHR,DSN=compware.CTL($67TDPRM)
The //STEPLIB DD points to the authorized step libraries for:
•
Enterprise Common Components
•
Abend-AID Common Components
•
Abend-AID for CICS
The //FDBDRPL DD points to the load libraries for:
•
Enterprise Common Components
•
Abend-AID Common Components
•
Abend-AID for CICS
The //FDBDCUST DD points to the Abend-AID Viewer customization file, that is
described in “4. Allocate, Copy and Upgrade, or Upgrade in Place a Customization File
