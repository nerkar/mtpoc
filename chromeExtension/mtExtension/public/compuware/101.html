Batch Dump Capture Address Space Customization
Chap 8
Note:
If your site does not plan to install batch Abend-AID, you may skip this chapter.
This chapter describes the steps in customizing the Abend-AID batch dump capture
address spaces (BDCAS). An Abend-AID BDCAS is an MVS address space that manages
batch dump capture processing. The BDCAS must be active whenever batch Abend-AID is
active. No batch Abend-AID reports will be generated, even to SYSOUT, unless you
configure and start the BDCAS. Compuware recommends that you run the BDCAS as a
started task that is always available. The BDCAS must be active before starting the Abend-
AID SVC51 interface.
You should have already started the batch dump capture address space (BDCAS) as
described in “7. Start the Batch Dump Capture Address Space” on page 4-6. Use the
information in this chapter to make changes to better meet your site’s requirements.
Your site must have at least one Abend-AID batch dump capture address space per
MVS image on which batch dumps are to be captured.
You can run multiple BDCASs
on a single MVS image, but doing so should not be necessary. You must repeat the steps
in this chapter for each BDCAS you want to create.
The following topics are described in this chapter:
•
Specifying the batch dump capture address space configuration parameters
($68BDPRM).
•
Modifying the batch dump capture address space JCL ($68BDCAS).
•
Verifying batch dump capture address space external security authority.
Specifying the BDCAS Configuration Parameters
($68BDPRM)
The parameters that control batch dump capture address space (BDCAS) configuration
are specified in a member or file that is pointed to by the BDCAS JCL //FDBDPARM DD
statement. The installation sample library (CTL) contains the sample BDCAS parameter
member ($68BDPRM).
Note:
The BDCAS parameter member does not have to reside in the installation sample
library (CTL). You can use any PDS member or sequential file for the parameters,
provided the dataset has the following attributes:
LRECL=80
BLKSIZE=any multiple of 80
RECFM=FB
You can also specify the BDCAS configuration parameters as SYSIN in the BDCAS JCL by
specifying
//
FDBDPARM DD * followed by the parameters.
This section describes specifying the BDCAS parameters, and contains subsections that
identify which parameters must be specified for each of the following configurations:
•
Required parameters that must always be specified
•
Optional tuning parameters.
