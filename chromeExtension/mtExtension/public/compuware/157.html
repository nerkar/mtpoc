This chapter describes the steps in customizing the Abend-AID for CICS transaction
dump capture address spaces. An Abend-AID for CICS transaction dump capture address
space (TDCAS) is an MVS address space that manages transaction dump capture
processing. The TDCAS must be active to take Abend-AID for CICS transaction dumps.
Compuware recommends that you run the TDCAS as a started task that is always
available.
You should have already started the transaction dump capture address space (TDCAS) to
run with the supplied defaults as described in “Starting the Transaction Dump Capture
Address Space” on page 5-3. Use the information in this chapter to make changes to
better meet your site’s requirements.
Your site must have at least one Abend-AID for CICS transaction dump capture
address space per MVS image on which transaction dumps are to be captured.
You
can run multiple TDCASs on a single MVS image, but doing so should not be necessary.
You must repeat the steps in this chapter for each TDCAS you want to create. Appendix
D, “Site Configuration Examples” provides examples of Abend-AID for CICS site
configurations.
The following topics are described in this chapter:

Specifying the TDCAS Space Configuration Parameters
($67TDPRM)
The parameters that control transaction dump capture address space (TDCAS)
configuration are specified in a member or file that is pointed to by the TDCAS JCL
//FDBDPARM DD statement. The installation sample library (CTL) contains the sample
TDCAS parameter member ($67TDPRM).
Note:
The TDCAS parameter member does not have to reside in the installation sample
library (CTL). You can use any PDS member or sequential file for the parameters,
provided the dataset has the following attributes:
LRECL=80
BLKSIZE=any multiple of 80
RECFM=FB
