Upgrading or Migrating to a New DB2 Release
Follow this procedure after you have upgraded or migrated a DB2 subsystem to a new
release. The batch Abend-AID product must be fully configured before beginning these
steps.
1.
See “Step 9. Configure Database Support” on page 6-1 for details regarding how to
access the DB2 support portion of the Abend-AID Installation and Customization
Dialog and how to access the Abend-AID Installation: Generated DB2 Support screen
(Figure 6-3 on page 6-5).
2.
Locate the entry on this screen for the subsystem that has been migrated to a new
DB2 release.
If an entry for the subsystem does
not
exist, do not continue with this procedure.
Instead, complete the procedure described in “Batch Abend-AID for DB2
Configuration Procedure” on page 6-5.
3.
Modify the migrated subsystem entry for the new DB2 release on the Abend-AID
Installation: Generated DB2 Support screen. Use the R (Replicate) line command to
copy the values of the subsystem entry and to display the Abend-AID Installation:
Site-specific DB2 Support screen (Figure 6-4 on page 6-6.) Review the displayed values
and make any required changes.
Modify the DB2 Release, DB2 libraries, and WLM environment name as appropriate.
You may also need to modify the specified Collection ID.
4.
Issue the GENERATE primary command to begin updating the installation JCL for the
migrated subsystem.
In the Dataset to Receive Generated JCL pop-up window that is displayed, specify the
name of the partitioned dataset that is to receive the generated JCL.
5.
Submit the newly generated JCL member $21
nxxxx
, where
xxxx
is the DB2 subsystem
name and
n
is the DB2 release number. This job rebinds the Abend-AID for DB2
product plan on the migrated subsystem and relinks the CWDB2OPT load module.
6.
You may optionally submit JCL members $82
nxxxx
, $92
nxxxx,
and $93
nxxxx
to
execute the Abend-AID for DB2 COBOL and PL/I installation verification programs.
7.
Enter the END primary command or press the PF3 key to return to the Abend-AID
Installation: Generated DB2 Support screen (Figure 6-3 on page 6-5).
Locate the entries for the migrated DB2 subsystem. An entry for both the previous
and the new DB2 release should be listed. Optionally delete the entry for the
previous release by issuing the D (Delete) line command.
8.
Enter the END primary command or press the PF3 key repeatedly to exit out of the
Abend-AID Configuration and Customization Dialog.
