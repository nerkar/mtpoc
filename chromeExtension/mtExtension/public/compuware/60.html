If you want to test batch Abend-AID further with your own jobs, note the following:
•
Before starting Abend-AID, run the program LMSINIT to establish the Runtime
License Management System.
•
To invoke Abend-AID for most jobs, a //SYSUDUMP or //SYSABEND DD statement
must be included in the job’s JCL. Language Environment jobs are an exception to
this requirement because of the way error processing is performed. See “Using Abend-
AID DD Statements” on page 9-2 for complete information about DD statements.
•
Error message AB5A2 may be issued if the Abend-AID load library is not accessible for
an application program failure being processed by Abend-AID. Error message AB62A
may be issued if the CSS load library is not accessible for an application program
failure being processed by Abend-AID. These messages indicate that Abend-AID could
not locate the next module to be loaded. For more information on how to make
these libraries accessible, see “Environmental Considerations” on page 11-6.
If you need to stop Abend-AID after testing, see “Abend-AID SVC51 Interface Installation
Utility (CWINSTAL)” on page E-4.
Validating Batch Abend-AID LE Support
Test the LE Support
Test the LE support by running JCL in the applicable member generated in Abend-AID’s
installation library:
•
$72LECOB: Tests diagnostics for COBOL programs.
•
$73LEPLI: Tests diagnostics for PL/I programs.
Testing Abend-AID Extended Language Support
Execute Installation Verification Programs
Execute one or more of the following members in the Abend-AID installation sample
library (CTL). These jobs compile listings to the source listing database and write the
Abend-AID report to the report shared directory. You can then use the Abend-AID Viewer
to view the abend report with source statements merged into the analysis.
