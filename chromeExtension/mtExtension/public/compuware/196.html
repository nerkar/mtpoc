Abend-AID Installation and Customization Guide
Abend-AID Datasets Verification
Note:
All Abend-AID, Abend-AID for CICS, and Abend-AID Viewer datasets that you
specify for the viewing server must be at the same product release. See
“Enterprise Common Components” on page 2-1 for the minimum Compuware
Shared Services and Host Communication Interface releases required for this
release.
The ddnames listed below point to dataset names that are tailored according to the
values you entered in the configuration and customization dialog. Verify them now to
ensure that they are correct:
//STEPLIB DD DISP=SHR,DSN=compware.kaz160.SKAZAUTH
// DD DISP=SHR,DSN=compware.lcx160.SLCXAUTH
//FDBDRPL DD DISP=SHR,DSN=compware.kaz160.SKAZLOAD
// DD DISP=SHR,DSN=compware.lcx160.SLCXLOAD
// DD DISP=SHR,DSN=compware.paa160.cust.load
// DD DISP=SHR,DSN=compware.paa160.SPAALOAD
// DD DISP=SHR,DSN=compware.kfx161.SKFXLOAD
//FDBDCUST DD DISP=SHR,DSN=aavwr.cust
//FDBDCLST DD DISP=SHR,DSN=aavwr.clist
//MSDDSECT DD DISP=SHR,DSN=aavwr.system.dsect
The //STEPLIB DD statement points to the following authorized step libraries:
•
Abend-AID Common Components
•
ECC.
The //FDBDRPL DD statement points to the following load libraries:
•
Abend-AID Common Components
•
ECC
•
Batch Abend-AID customization
•
Batch Abend-AID
•
Abend-AID for CICS.
The //FDBDCUST DD statement points to the Abend-AID Viewer customization file,
which is described in “4. Allocate, Copy and Upgrade, or Upgrade in Place a
Customization File ($62CUST)” on page 3-6.
The //FDBDCLST DD statement points to the CLIST libraries that are used to link
TSO/ISPF-only viewing access to the Compuware Shared Services utilities, File-AID, and
the HOTKEY command. See “Allocating Libraries for TSO/ISPF-Only Viewing Access
Facilities” on page 16-3 for more information.
The //MSDDSECT DD statement points to the system DSECT file. See “3. Define the MVS
DSECT Images ($61MVSD)” on page 3-6 for more information.
