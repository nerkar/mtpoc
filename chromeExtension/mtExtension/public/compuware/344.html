ACF2 Release 5.2 and Less Current
ACF2 release 5.2 and less current require SAFPROT rules to enable the Abend-AID Viewer
to use the SAF security API. The specific rules you should write are shown below. In the
examples,
aavwr
is a unique qualifier, used to ensure that the rule keys are unique.
SYS1 / SAFPROT.aavwr CLASSES(-) CNTLPTS(FDB-) SUBSYS(FDB-)
As an alternative, you can write the following rules:
SYS1 / SAFPROT.aavwr1 CLASSES(VERIFY) CNTLPTS(FDB-) SUBSYS(FDB-)
SYS1 / SAFPROT.aavwr2 CLASSES(DATASET) CNTLPTS(FDB-) SUBSYS(FDB-)
Note:
If you choose the alternative method from the two methods shown above, you
need to write a third SAFPROT rule if
both
of the following are true:
–
You are restricting access to Abend-AID Viewer functions, as described in
“Controlling User Access to Specific Functions” on page 23-8
.
–
You specify something other than the default value, DATASET, for the
EXTERNAL_SECURITY_RESOURCE_CLASS viewing server configuration
parameter, as described in “EXTERNAL_SECURITY_RESOURCE_CLASS=class” on
page 15-10.
Enabling RACROUTE under Top Secret
If you use Top Secret, define the viewing server’s user ID as a multi-user facility.
