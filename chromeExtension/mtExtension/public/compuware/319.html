21-7
Creating Region Dump Action Definition Entries
Note:
This option is available only if Abend-AID for CICS is installed on this viewing
server.
Region dump action definition entries specify which TSO user or users are notified and
what contact information is provided when region dumps occur. The Region Action
Definitions screen supports a maximum of 8,298 entries.
Note:
The Abend-AID for CICS MVS post-dump exit is required to use TSO notification
for region dumps taken to a SYS1.DUMPxx, a user-defined SDUMP dataset, or an
automatically allocated dataset. “MVS Post-Dump Exit” on page 17-3 describes
installing the MVS post-dump exit.
To create region dump action definition entries, complete the following procedure:
1.
Display the Region Action Definitions screen, shown in Figure 21-10, using one of
two methods:
–
Tab to the Region Dump Action Definitions option on the Action Definition
menu (Figure 21-2 on page 21-2), and then press Enter.
–
Type
ACTREGN

=3.3
in the COMMAND (or OPTION) field on any
customization

unless you have accessed the customization facility.

Region Action Definitions Screen
The Region Action Definitions screen initially displays the wildcard character (*) in
the CICS Job Name, CICS APPLID, MVS SYSID, and Abend Code fields. These fields
are used to qualify the dumps for which you want notification.
2.
Enter your configuration values in the appropriate columns.
Notes:
a.
A default region action definition is not provided and you cannot create one
because when a region dump occurs, Abend-AID for CICS scans all of the region
action definitions with no regard to order or hierarchy.
b.
A single asterisk (*) can be used as a wildcard character to partially qualify any
CICS job name, CICS APPLID, MVS SYSID, or abend code. An asterisk entered as a
full qualifier means that the user indicated in the TSO ID field is notified of
dumps for
all
matching job names, APPLIDs, and so forth.
c.
To notify multiple users of a single dump occurrence, create an action definition
entry for each user.
d.
Abend-AID for CICS submits a batch job that uses the MVS SEND command to
notify TSO users. If your security system has the OPERCMDS class activated, you
may need to authorize access to the SEND command.
