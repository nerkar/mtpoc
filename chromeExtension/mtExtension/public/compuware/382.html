Abend-AID Installation and Customization Guide
Abend-AID SVC51 Interface Installation Utility
(CWINSTAL)
With the CWINSTAL utility, you start and stop Abend-AID and perform various other
functions related to the Abend-AID SVC51 interface. The JCL required to execute
CWINSTAL can be found in member $12INSTL in the Abend-AID installation library.
Notes:
1.
The Abend-AID batch dump capture address space (BDCAS) must be active before
starting the SVC51 interface. If you haven’t already started the BDCAS and an Abend-
AID subsystem, see “7. Start the Batch Dump Capture Address Space” on page 4-6 for
more information.
2.
When testing a new release of Abend-AID, you must use the new Abend-AID SVC51
interface, the new Abend-AID SMP/E-managed load library, the new Abend-AID
customization load library, and the appropriate CSS load library. See “Enterprise
Common Components” on page 2-1 for the appropriate CSS release.
3.
Beginning with Abend-AID Release 11.1, CWINSTAL installs the ABENDAID hook
into the dynamic Extended Link Pack Area (ELPA). In prior releases, the hook was
installed into ECSA.
If you use the Release 11.1 CWINSTAL to install the ABENDAID
hook for Release 10.1 or 10.2, you must first apply current maintenance to the
release to prevent a possible S0C4. See “Enhanced CWINSTAL Utility” on page 11-6
for the specific PTF number and required maintenance level.
All Abend-AID diagnostics, including those for LEAID and SNAP-AID processing, require
the use of the Abend-AID SVC51 interface.
The SVC51 interface operates in production mode only, in test mode only, or in both
production and test modes simultaneously. Only one production SVC51 interface and/or
one test SVC51 interface can be in operation at a time. If Abend-AID is to be started in
both production and test modes, the production SVC51 interface must be started first
and the test SVC51 interface must be started second. This enables the test SVC51
interface to gain control first and perform its job name evaluation. If the job name does
not match a CWINSTAL TEST parameter job name mask, then control is passed to the
production SVC51 interface.
You start Abend-AID production mode, which diagnoses failures in all jobs, by executing
CWINSTAL without specifying a parameter. You start test mode, which diagnoses failures
in specified jobs, with the TEST parameter. Invoke other functions with the parameters
listed below. Messages generated from CWINSTAL appear in the job log.
CWINSTAL syntax
PARM=‘LIST’
‘TEST,jobname...’
‘STOP’
‘STOP,*’
‘AJOB,jobname...’
‘DJOB,jobname...’
Parameters
LIST
Displays the status of the Abend-AID SVC51 interface (the releases started and other
pertinent status information).
TEST,jobname
Starts a test version of Abend-AID for selected jobs. The production version of Abend-
AID does not need to be stopped and restarted in order to start a test version of
Abend-AID.
