Allocating the Viewing Server Datasets
Each viewing server has datasets that it
owns
. You must allocate one of each of the
datasets described in this section for each viewing server you create (allocating an IPCS
directory is optional).
The viewing server datasets are:
•
Dump information file
•
IPCS directory (optional)
•
Persistent data file
•
Viewing server work file
•
Shared directory
•
Report database(s)
•
Viewing server Scratchpad listing file
Note:
If you’re using a VSAM buffer management package, exclude the above Abend-
AID viewing server datasets so they’re not managed by the package.
Except for the shared directory, report databases, and the Scratchpad listing file, you
cannot share these datasets with other viewing servers.
Note:
When sharing a shared directory among multiple servers that are used to view
system dumps, update the viewing server configuration parameter
"SHARED_DIRECTORY_OPTIONS=UNSHARE|share" to SHARE to support this
functionality. See page 15-13 for more information.
The jobs to allocate these datasets are located in the installation sample library (CTL) in a
partially tailored format. The viewing server dataset names were not tailored when the
allocation jobs were generated, because your site may have multiple viewing servers and
therefore multiple sets of the viewing server datasets. However, the Abend-AID product
dataset names (customization file, and so forth) were generated into the sample JCL.
