Authorizing the MSDCCOPY Procedure
If you install the Abend-AID Viewer post-dump exit, you must associate the MSDCCOPY
procedure name with a suitably authorized user.
Be aware that MSDCCOPY must have authority to create dump-copy datasets whose
names are specified in the region dump capture profile options. For example, if
MSDCCOPY is associated with user AAVIEWER and the region dump capture profiles
specify that all copied dump datasets are copied to datasets beginning with the high-level
qualifier SDUMP, then AAVIEWER must be granted authority to create ‘SDUMP.nnnn...’
datasets.
