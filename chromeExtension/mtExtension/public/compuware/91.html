Concurrent DB2 Releases
You can run the Abend-AID for CICS Release 16.05 DB2 option concurrently with
previous currently supported Abend-AID for CICS releases in a single DB2 subsystem.
Follow the appropriate guidelines below, depending on whether you want to share the
Abend-AID for CICS DB2 CSD entries.
•
Sharing the RDO entry between Release 16.05 and a previous release:
The default
transaction ID for the Abend-AID for CICS DB2 option is AADB. You can continue to
use this transaction ID for the previous release, and create a new, unique transaction
ID for Release 16.05. Bind the Abend-AID for CICS DB2 option Release 16.05 plan
with a name other than the plan name specified for the previous release, and add an
entry to the CICS CSD for the new Release 16.05 transaction ID pointing to module
CTCCxD22, where x is a letter representing the CICS release:
–
N —
CICS Transaction Server for z/OS 5.3 (CICS 7.0)
–
M
—
CICS Transaction Server for z/OS 5.2 (CICS 6.9)
–
L
—
CICS Transaction Server for z/OS 5.1 (CICS 6.8)
–
K —
CICS Transaction Server for z/OS 4.2 (CICS 6.7)
–
Z —
CICS Transaction Server for z/OS 4.1 (CICS 6.6)
Ensure that you use a separate GRPLIST for each release of Abend-AID for CICS if you
are running concurrent releases of the DB2 option and sharing the RDO entry.
•
Using a new RDO entry:
If you intend to create a new RDO entry for Release 16.05,
complete the instructions for installing the DB2 option, as described in “Abend-AID
for CICS for DB2 Configuration Procedure” on page 6-8. Ensure that you bind the
plan for Release 16.05 with a name other than the plan name specified for the
previous release.
