Implementing the REXX API
The Abend-AID REXX application program interface (API) lets you process REXX EXECs
against batch, CICS transaction, and region dumps. You load EXECs from the SYSEXEC
DD specified in the Abend-AID viewing server JCL.
Step 1. Modify Viewing Server JCL
If you want to be able to execute REXX EXECs from within Abend-AID, include in your
viewing server JCL the following DD statement:
//SYSEXEC DD DSN=compware.kaz160.SKAZREXX
This dataset contains the Abend-AID for CICS-supplied REXX sample, SAMPREXX, and
the REXX EXECs you write, as described in the
Abend-AID for CICS User’s Guide
. The
SKAZREXX dataset is a fixed block dataset with a logical record length of 80. You can
concatenate multiple datasets on this DD, but they all must have the same record format
and logical record length.
Step 2. Stop and Restart the Viewing Server
After you complete the previous step, you must stop and then restart the viewing server.
Note:
You can restrict access to the REXX API by coding the REXX subparameter of the
EXTERNAL_SECURITY_FUNCTION_CHECK viewing server configuration
parameter. Refer to “Parameters to Enable External Security for Viewing Servers”
on page 15-8 for more information.
Adding Databases to a Report Shared Directory
Refer to the
Compuware Shared Services User/Reference Guide
for information about using the
online CSS Utilities or the batch file utility CWDDALLU to create report/transaction
databases and attach them to a shared directory. This guide is provided in PDF format on
the Enterprise Common Component (ECC) product documentation site on Compuware’s
FrontLine website at
http://go.compuware.com.
You can add Abend-AID report or CICS transaction databases to the report shared
directory (MFDCATLG) while the viewing server is running. The new database(s) may not
appear in the viewing server for some time. For immediate confirmation, recycle the
viewing server task.
Abend-AID starts using the new database(s) immediately unless you have assigned
specific databases to batch jobs, IMS, or CICS regions. If you have assigned specific
databases, you need to update those assignments. For batch and IMS, see “Modifying
Report Database Groups” on page 20-53. For CICS, see “Assigning CICS Transaction
Databases to CICS Regions” on page 20-12.
