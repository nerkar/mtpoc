Transaction Dump Capture Address Space Customization
12-5
Note:
If you specify the LU2_APPLID parameter for VTAM access, you must also
specify ENABLE_IUR=YES.
ENABLE_IUR=YES|no
Indicates whether the interactive user region (IUR) application starts in the
transaction dump capture address space. The IUR enables you to log onto the TDCAS
to perform basic commands such as shutting down the TDCAS. The default is
YES
. If
you delete the LU2_APPLID parameter, specify
NO
for this parameter.
EXTERNAL_SECURITY_ENABLED=NO|yes
Specifies whether Abend-AID for CICS requires a password when a user signs onto the
TDCAS.
NO
Abend-AID for CICS does not require a password. This value is the default.
YES
Abend-AID for CICS external security checking is enabled, and Abend-AID for
CICS requires users to specify a user ID and password when logging onto the
TDCAS.
MANAGEMENT_REPORTING=NO|yes
Specifies whether to enable Abend-AID Fault Analytics for the associated CICS
region(s). Abend-AID Fault Analytics is Compuware’s fault management tool that
captures Abend-AID for CICS fault data to a reporting database. If your site has
installed this product, specify
YES
. The default is
NO
. If your site plans to use Fault
Analytics to access Abend-AID for CICS diagnostics via the Internet, see “Parameters
to Support the Abend-AID Web Browser Interface, Fault Analytics, and TSO ISPF
Viewing Access via TCP/IP” on page 15-6.
Note:
This parameter enables only Abend-AID for CICS information to be sent to
Abend-AID Fault Analytics. To enable Abend-AID Viewer information to be
sent to Abend-AID Fault Analytics, see the Abend-AID Viewer parameter
“MANAGEMENT_REPORTING=NO|yes” on page 15-13. To enable batch
Abend-AID information to be sent to Abend-AID Fault Analytics, review
installing module CWAFPUBL in the Abend-AID Fault Analytics installation
guide.
TRACE=size
Specifies the size of the internal trace buffer in K-bytes. Valid values are 256 through
32768. The default value is
1024
.
DFHRPL_READ=YES|no
Specifies where CSECT and load module information is processed during transaction
dump capture.
YES
CSECT and load module information is processed in the transaction dump
capture address space. This option is accomplished by having the TDCAS allocate
and open the datasets containing the load modules. For performance reasons,
these datasets remain opened in the TDCAS until the region is shutdown. This
value is the default.
NO
CSECT and load module information is processed in the CICS region. This option
was the process used in Abend-AID for CICS Release 4.3 and less current.
Note:
To temporarily enable/disable the DFHRPL_READ function via the MVS
MODIFY command, see “Controlling Opening and Closing of Load Libraries”
