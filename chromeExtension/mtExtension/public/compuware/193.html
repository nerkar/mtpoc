MANAGEMENT_REPORTING=NO|yes
Specifies whether to enable Abend-AID Fault Analytics. Abend-AID Fault Analytics is
Compuware’s fault management tool that captures Abend-AID fault data to a
reporting database. If your site is licensed for and has installed this product, specify
YES
. The default is
NO
. If your site plans to use Fault Analytics to view Abend-AID
diagnostics via the Internet, see “Parameters to Support the Abend-AID Web Browser
Interface, Fault Analytics, and TSO ISPF Viewing Access via TCP/IP” on page 15-6.
Note:
This parameter enables only Abend-AID Viewer information to be sent to
Abend-AID Fault Analytics. To enable Abend-AID for CICS information to be
sent to Abend-AID Fault Analytics, see the Transaction Dump Capture
Address Space parameter “MANAGEMENT_REPORTING=NO|yes” on page
12-5. To enable batch Abend-AID information to be sent to Abend-AID Fault
Analytics, review installing module CWAFPUBL in the Abend-AID Fault
Analytics installation guide.
TRACE=size
Specifies the size of the internal trace buffer in K-bytes. Valid values are 256 through
32768. The default value is
1024
.
HEALTH_CHECK=YES|no
Specifies whether health checks using the IBM Health Checker for z/OS are started
when the viewing server is started. The health checks provide information about
storage utilization. The default is
YES
. See Appendix H, “Abend-AID Health Checker”
for more detailed information.
Parameters to Integrate with DevEnterprise
The following parameters are required to integrate DevEnterprise Release 5.1 or more
current impact analysis into the Abend-AID/Abend-AID for CICS Diagnostic Summary in
the Abend-AID Viewer/Abend-AID Web Browser Interface.
DE_SERVNAME
Specifies the maximum 64-character, fully qualified workstation name where the
DevEnterprise server is running.
DE_PORT
Specifies the DevEnterprise listening port number. Valid values are 1024 through
65535.
Parameter to Restrict the HOTKEY Command Users
The following parameter is required for batch Abend-AID sites that want to restrict
HOTKEY command users from Abend-AID Directory access.
HOT_KEY_ACCESS_RESTRICTED=YES|no
Specifies whether the HOTKEY command users are restricted. If
YES
is specified, users
accessing the viewing server via the HOTKEY command will not be able to view
Abend-AID Directory entries using the DIRECTORY command. They will have access
to their own jobs only, one-at-a-time. The default is
YES
.
