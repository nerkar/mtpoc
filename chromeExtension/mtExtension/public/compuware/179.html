Viewing Server Customization
Chap 15
A viewing server is an MVS address space. The viewing server does not need to be active
to
take
dumps, but it must be active to view any dump information or to import SVC
dumps. Compuware recommends that you run the viewing server as a started task that is
always available.
Your site must have at least one Abend-AID viewing server, although you can run
multiple viewing servers if it better suits your needs. You must repeat the steps in
Chapter 3, “Abend-AID Viewer Configuration” for each viewing server you want to
create. For the initial installation, Compuware recommends you create only one viewing
server. You can easily add additional viewing servers after you are more familiar with the
Abend-AID Viewer.
Each viewing server has datasets that it
owns
. Except for the shared directory and the
Scratchpad listing files, these datasets cannot be shared between servers. The viewing
server datasets are described in “Allocating the Viewing Server Datasets” on page 3-5.
If you are licensed for both Abend-AID and Abend-AID for CICS, the steps for using one
viewing server for both batch and CICS transaction reports are also described in “Sharing
a Viewing Server with Abend-AID for CICS” on page 15-21. Other Abend-AID Viewer
configurations are described in Appendix D, “Site Configuration Examples”.

Specifying the Viewing Server Configuration
Parameters ($64xxxx)
The parameters that control viewing server configuration are specified in a member or
file that is pointed to by the viewing server JCL //FDBDPARM DD statement. The
installation sample library (CTL) contains two sample viewing server parameter
members:
•
$64ISPF:
A sample configuration to support TSO/ISPF viewing access and/or CICS
local
viewing access.
•
$64VTAM:
A sample configuration to support VTAM viewing access. This is the
default parameter member on the //FDBDPARM DD statement in the default server
JCL (see “Viewing Server Parameters Dataset” on page 15-18).
