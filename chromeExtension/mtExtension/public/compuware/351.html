Appendix A.
A
Controlling the Abend-AID Viewing Server, BDCAS,
and TDCAS
App A
This appendix describes starting and stopping Abend-AID viewing servers, batch dump
capture address spaces (BDCASs), transaction dump capture address spaces (TDCASs) and
closing/reopening their FDBDLOG files. It also describes how to display information
about a viewing server or TDCAS.
Starting the Viewing Server, BDCAS and TDCAS
You can start a viewing server, BDCAS, or TDCAS in one of two ways, depending on the
way you configured your viewing server, BDCAS, or TDCAS JCL:
•
Submit the viewing server, BDCAS, or TDCAS job.
•
Start the PROC that initiates the viewing server, BDCAS, or TDCAS.
If the viewing server, BDCAS, or TDCAS was generated as a PROC, Compuware
recommends that you add a START command for the viewing server, BDCAS, or TDCAS to
SYS1.PARMLIB(COMMNDxx) so that the viewing server is automatically started and
available after a system IPL. Include the REUSASID=YES parameter with the START
command. Using this parameter prevents the IBM IEF352I message from being produced
at shutdown of the viewing server, BDCAS, or TDCAS.
Notes:
1.
An Abend-AID subsystem must be active before you can start the BDCAS or TDCAS.
2.
If you run the viewing server, BDCAS, or TDCAS as a job and a user is logged onto it
when the viewing server, BDCAS, or TDCAS stops or is shutdown, you may get one of
the following messages:
IEF355A INITIATOR TERMINATED, RESTART INITIATOR.
IEF352I ADDRESS SPACE UNAVAILABLE.
This condition is normal and occurs because the viewing server, BDCAS, or TDCAS is
using cross-memory services. Running the viewing server, BDCAS, or TDCAS as a
started task avoids this condition.
Displaying Information about a Viewing Server, BDCAS,
or TDCAS
To display general information about a viewing server, BDCAS, or TDCAS, enter the
SERVINFO
command in the COMMAND (or OPTION) field on any Abend-AID Viewer
screen and press Enter. The Server Information screen is displayed, listing users currently
logged onto the viewing server or the user ID assigned to the regions attached to the
batch or transaction dump capture address space, as well as Abend-AID release
information.
