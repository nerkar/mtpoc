//IPCSDDIR DD Statement
Ensure that the //IPCSDDIR DD statement points to the IPCS directory that you allocated
in “6. Allocate the Optional IPCS Directory ($63IPCS)” on page 3-7.
//SYSTSIN DD Statement
Ensure that the //SYSTSIN DD statement points to the Abend-AID Viewer target sample
library (SKAZSAMP) member MSDSENDC, as follows:
//SYSTSIN DD DISP=SHR,DSN=compware.kaz160.SKAZSAMP(MSDSENDC)
//SYSPROC DD Statement
Ensure that the //SYSPROC DD statement points to the Abend-AID Viewer target sample
library (SKAZSAMP). You can concatenate other libraries to the SYSPROC statement if you
want to execute IPCS CLISTs in those libraries from within the Abend-AID Viewer.
//SYSTSPRT DD Statement
Ensure that the //SYSTSPRT DD statement is assigned to an appropriate JES output class
with the following DCB parameters:
//SYSTSPRT DD SYSOUT=*,DCB=(RECFM=VBA,LRECL=0,BLKSIZE=140)
//MFDDPxxx DD Statements
Valid for Abend-AID for CICS only. The sample viewing server JCL also contains one to
four DD statements (MFDDPxxx) that point to the CICS load libraries that contain the
DFHPDX (or DFHPDxxx) modules that you included in the configuration and
customization dialog, where xxx refers to the CICS release. When an Abend-AID Viewer
user executes the IPCS CICS VERBEXIT (DFHPDX), Abend-AID loads the DFHPDX load
module appropriate to the CICS release of the dump from the corresponding CICS load
library.
Note:
If you are not supporting IPCS access from the Abend-AID Viewer, you can delete
or comment out the following DD statements from the viewing server JCL:
//IPCSDDIR
//SYSTSIN
//SYSPROC
//SYSTSPRT
//MFDDPxxx
Compuware Source Listing DSNs
Compuware recommends that you include Compuware source listing datasets at Abend-
AID Viewer start-up. Specify them on the //MFDDXLSF DD statement. You can
concatenate several datasets on the //MFDDXLSF DD statement.
If you’re installing Abend-AID for CICS, you can define source listing files in the viewing
server JCL or to the CICS region.
•
Defining them in the viewing server JCL makes them available to all reports in the
viewing server.
•
Defining them to the CICS region makes them available to the reports for that
region. For more information about defining them to the CICS region, see “Source
Listing Files” on page 5-5.
