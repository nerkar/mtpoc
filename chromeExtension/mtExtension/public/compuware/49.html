5. Allocate the Dump Information File ($63INFO)
The dump information file is a VSAM KSDS. You must allocate one dump information file
per Abend-AID viewing server, and you cannot share dump information files between
viewing servers.
Member $63INFO in the installation sample library (CTL) contains a job to allocate the
dump information file using IDCAMS. The job then initializes the file using an Abend-
AID utility. Before you run $63INFO, make the following modifications:
1.
Change each occurrence (3) of the file dataset name to the name you choose for the
file.
2.
If applicable, verify that the volume serial number or SMS storage and management
class conform to your site standard.
3.
Verify that the //STEPLIB DD statement points to a concatenation of your Abend-AID
Common Components, ECC, and Abend-AID/Abend-AID for CICS target
nonauthorized load libraries (SKAZLOAD, SLCXLOAD, and SPAALOAD/SKFXLOAD),
in that order.
6. Allocate the Optional IPCS Directory ($63IPCS)
Note:
The IPCS directory is an optional VSAM KSDS. If you want to be able to execute
IPCS commands when logged onto this Abend-AID viewing server, you must
allocate the IPCS directory specifically for your operating system.
The IPCS directory used by Abend-AID is a standard IPCS directory. There is one IPCS
directory per Abend-AID viewing server, and you cannot share IPCS directories between
viewing servers. Further, do not access the viewing serverâ€™s IPCS directory outside of
Abend-AID.

IPCS directory using IDCAMS. The job then initializes the file using an Abend-AID utility.
Before you run $63IPCS, make the following modifications:

Common Components, ECC, and Abend-AID/Abend-AID for CICS target
nonauthorized load libraries (SKAZLOAD, SLCXLOAD, and SPAALOAD/SKFXLOAD),
in that order.
