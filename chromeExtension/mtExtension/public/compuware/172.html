Using CEECDATX
The IBM-supplied abnormal termination exit for CICS, CEECDATX, causes a transaction
dump to be taken for an unhandled condition of severity 2 or higher. This exit causes a
transaction dump by issuing an EXEC CICS DUMP TRANSACTION DUMPCODE(’4039’)
TASK. If you use CEECDATX, Compuware also recommends issuing an EXEC CICS
IGNORE CONDITION SUPPRESSED prior to the EXEC CICS DUMP TRANSACTION to
avoid an AEXW abend (LE shows as a 4087 abend) when suppressing IBM transaction
dumps
No action is necessary to install CEECDATX because it is linked when LE is installed. The
exit is active unless disabled by specifying the
4039
dump code when issuing the CEMT
transaction. By doing so, the default transaction dump will not be generated.
For information about using an LE abnormal termination exit, refer to the IBM
document,
Language Environment for z/OS Customization.
Specifying TERMTHDACT(UAONLY)
Specify the LE run-time option TERMTHDACT(UAONLY) in the installation LE options
member CEECOPT or the user LE options member CEEUOPT. UAONLY is an option that
creates a CICS transaction dump when an unhandled condition of severity code 2 or
greater occurs. UAONLY is the preferred option if one is specified for TERMTHDACT.
LE
will generate a CICS transaction dump.
Note:
Compuware does not recommend specifying the TERMTHDACT(UATRACE)
option because not as much information is available to be captured.
Compuware’s Recommendations
Compuware recommends installing the CEEWUCHA condition handler as the preferred
method for these reasons:
•
Using the CEEWUCHA condition handler requires less overhead.
•
The Abend-AID for CICS transaction dump global option
LESUPPORT=Y is not
required. (LESUPPORT=N is the default.) For CICS-detected errors such as ASRA, this
condition handler does not cause a second CICS transaction dump to be issued as it
does for the CEECDATX exit or the TERMTHDACT(UAONLY) option.
•
LE does not convert the abend code to a 4039 but keeps the original abend code; for
example, the COBOL 1nnn abend.
The use of the CEECDATX exit or the TERMTHDACT(UAONLY) option can cause
multiple transaction entries for a single occurrence of an abend for CICS-detected errors.
The Abend-AID for CICS transaction dump global option LESUPPORT=Y is required to
suppress the duplicate Abend-AID for CICS transaction entry.
