Abend-AID Installation and Customization Guide
Customers supporting large operations tend to choose one of two basic configurations,
depending on how the site normally manages software migration (in particular, how
CICS is managed). The two basic multiple-MVS image configurations are:
•
Running one Abend-AID viewing server per MVS image, with each viewing server
configured to support all of the CICS regions running on that image. Sites that use
this setup usually configure their MVS images as identically as possible for ease of
support and migration of new software throughout their environment. “Multiple
Viewing Servers Sharing Files Across MVS Images” on page D-8 illustrates this type of
configuration.
Some of these customers run multiple viewing servers on each MVS image for
performance reasons, or to segregate dump information for their CICS regions (for
example, production from test). The common thread is that they do not share
viewing servers across MVS images.
•
Running one “central” viewing server that supports several (or all) MVS images.
These sites require a central point of control for all of their MVS images. “One
Viewing Server Supporting Multiple MVS Images” on page D-11 illustrates this type
of configuration.
Running with Multiple TDCASs on a Single MVS Image
One Abend-AID transaction dump capture address space (TDCAS) is required on every
MVS image on which you will capture CICS transaction dumps. Although only one
TDCAS is required per MVS image, you can run more than one if it better fits into the
way your site manages Abend-AID. For example, if you require completely segregated
Abend-AID environments to support certain CICS regions, you will probably want to
create additional TDCASs.
Some customers may want to create a new TDCAS when they migrate to a new release of
Abend-AID. This should not normally be necessary, as the TDCAS is designed to be
downward compatible. The recommended procedure when upgrading to a new Abend-
AID release is to stop your current TDCAS, update the TDCAS JCL to point to the libraries
for your new release of Abend-AID, and restart the TDCAS. All CICS regions, even those
you have not yet upgraded to the new Abend-AID release, will use the upgraded TDCAS.
If you are not comfortable with this migration path, you need to create a second TDCAS
to support the new Abend-AID release.
If you decide you require multiple TDCASs on an MVS image, you need to specify which
CICS regions should be processed by each TDCAS. Use the CICS Region Configuration
screen, described in “Configuring Regions” on page 20-3, to perform this function.
