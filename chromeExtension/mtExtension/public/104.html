WRITE_SMF=(NO|yes,SMF record number)
Specifies whether to write out SMF records containing information for each abend at
the time of dump capture for batch reporting. Customers with a suitable reporting
process can use these records to produce reports on abend activity. This parameter
has been provided to enable the writing of these SMF records and to allow the user to
specify the SMF record number.
NO
Do not write an Abend-AID SMF dump capture record. This is the default.
YES
Write an Abend-AID SMF dump capture record.
SMF record number
The SMF record number to be used by Abend-AID. This number can be any
available record number between 128 and 255, but it must not be the same
number that is used by any other z/OS software.
Member SMFDSECT in the Abend-AID installation sample library (CTL) contains an
Assembler DSECT layout for the Abend-AID SMF record.
Parameters Required to Support MVS Automatic Restart Manager
Abend-AID can register to be restarted by the MVS automatic restart manager (ARM). The
following parameters are required for Abend-AID batch dump capture address spaces to
be restarted by ARM in conjunction with the site-defined ARM policies.
ARM_ENABLED=NO|yes
Specifies whether the batch dump capture address space is to be registered for restart
with ARM. The default is
NO
.
ARM_ELEMENT_SUFFIX=bdcasname|suffix
Specifies the one- to eight-character suffix of the ARM element name used to register
with ARM. The element name used by batch Abend-AID to register with ARM is
KMPssidsuffix, where
KMP
is a constant,
ssid
is the four-character MVS system ID, and
suffix
is the ARM_ELEMENT_SUFFIX. The element name is used by ARM to select a
restart policy. Each Abend-AID BDCAS using ARM requires a unique suffix. Unless
you specify a different suffix, the BDCAS uses the default, the BDCAS name
parameter value, which is the one- to eight-character BDCAS name specified in the
BDCAS execute statement.
Note:
If the automatic restart manager (ARM) restarts a job, JES discards all non-spin
SYSOUT datasets created during the previous execution. SPIN=UNALLOC is
automatically included on the //FDBDLOG DD statement for the SYSOUT
datasets in the BDCAS generated JCL to avoid losing valuable debugging
information. For example:
//FDBDLOG DD SYSOUT=*,SPIN=UNALLOC
Modifying the BDCAS JCL ($68BDCAS)
Installation sample library (CTL) member $68BDCAS contains sample startup JCL for the
Abend-AID subsystem and batch dump capture address space. The Abend-AID subsystem
is defined by the MVS_SUBSYSTEM viewing system parameter, as described in
“MVS_SUBSYSTEM=(name,OWNER|USER)” on page 15-4. Review and modify the sample
JCL according to the guidelines in this section.
